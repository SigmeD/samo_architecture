## M1 — `CONV-RBAC-DNM-001`: маркетолог вне RBAC-матрицы

- **id:** `CONV-RBAC-DNM-001` v1.4 · **тип:** ADD
- **Канон сейчас:** матрица RBAC ДНМ — **10 ролей** (Уч, Род, Кур, СК, КФ, Адм, Мен, Фин, Фрч, Рук). **Маркетолога нет**, хотя роль `op-marketolog-dnm` есть в реестре (`CONV-ROLES-DNM-001` v1.3, placeholder 06.06) и описана в `REG-DNM-MARKETING-001` v1.1.
- **Предложение:** добавить колонку **Мрк** в RBAC-матрицу. Черновик доступов (по проектному доку):
  - Реклама / SMM / лидогенерация — `CRU` (свой закреплённый аккаунт);
  - Карточка лида: источник/UTM — `CRU`; close rate (от продаж) — `R`;
  - Ассеты HQ — `R` (локализация в рамках бренда); брендбук/центральный контент — `—`;
  - Финансы / договор / оплата — `—`; выручка (для ROAS) — `R` агрегат (Фаза 3, конфиденц.);
  - Аналитика — только **закреплённый аккаунт** (HQ-маркетолог — все аккаунты сети).

## M2 — Сквозной пайплайн атрибуции (несущая ось)

- **id:** `ARCH-SERVICEMAP-001` v1.1 / `SPEC-CRM-TZ-001` v1.1 · **тип:** ADD (кросс-кабинетная архитектура)
- **Намерение (Google Doc 2 §0–§3):** один лид = метка источника (UTM), которая **не теряется** до оплаты;
  **точка истины конверсии = подтверждение оплаты бухгалтером** → одним событием обновляет close rate маркетолога
  (MK2), воронку продаж (RP4), рост франчайзи (D2). Контур: маркетолог → CRM → менеджер → администратор → бухгалтер,
  и **обратно** close rate.
- **Предложение:** зафиксировать единый лид-ID + сквозной источник как архитектурное требование (несущая ось
  маркетинговой аналитики). Без него (R1) маркетолог слеп — остаётся на CPL/охватах. Фазирование: MVP (захват
  источника + воронка + базовый close rate) → Фаза 2 (авто-атрибуция, A/B) → Фаза 3 (ROAS с фидом выручки).
- **Связь:** WhatsApp-лиды (омниканальный дожим менеджера, кабинет sales §9) тоже должны попадать в контур с источником.

## M3 — KPI маркетолога = качество, не объём (R2)

- **id:** `SPEC-KPI-PAYOUT-001` v1.1 / материалы обучения · **тип:** CHANGE
- **Намерение (Google Doc §14 R2, Doc 2 §4):** в обучении KPI заданы как «число лидов / стоимость лида» — это
  премирует **объём**, а не качество. Первичный показатель должен быть **квалифицированные лиды / close rate**;
  объём лидов и охваты — диагностические. Иначе дашборд (качество) и премия (объём) тянут в разные стороны.
- **Предложение:** в системе премирования/скоркарте маркетолога первичный KPI = close rate / квалиф. лиды.

## M4 — Валидация/дедуп входящих лидов (R3)

- **тип:** ADD · **Намерение (Google Doc §14 R3):** платный лидген — риск ботов/фейков; нужна валидация и дедуп
  входящих лидов на входе в воронку. В каноне не специфицировано.
