## TL;DR

Редизайн **накладывает** на канон ученика три приоритета (удержание · посещение · честная механика баллов) и когортный слой JUNIOR/SENIOR, **не ломая инварианты** (ДЗ без баллов, тесты 70%/10, блокирует только итоговый, видео-заранее, СУ≠проект, финансы скрыты). Главные расхождения — в механике соларов/рейтинга: разрез **merit vs gift**, merit-достижения в рейтинг **сверх потолка 1190**, подача ученику **тирами/перцентилем** вместо строгого топа, когортная нормировка лидерборда. Посещение и удержание — **аддитивные** наложения (раннее предупреждение, двусторонний канал родителю, наставник посещаемости, мотивация-pull). Всё — **на согласовании**.

## U1 — `SPEC-DNM-RATING-001` §2: метрика рейтинга ученика = Версия A
- **id:** `SPEC-DNM-RATING-001` v1.2 §2 (`OQ-RATING-01`, метрическая часть) · **тип:** RESOLVE
- **Канон сейчас:** конфликт двух источников — **Версия A** (`tz-dnm` §18.5 / `M3` FR-M3-150: посещаемость + своевременность ДЗ + солары) vs **Версия B** (`M16` FR-M16-009: пройденные уроки + средний балл тестов + звёзды + баланс + % ДЗ). `OQ-RATING-01` открыт.
- **Решение владельца (09.06):** канонична **Версия A** (3 показателя). 5-показательную M16-версию из рейтинга ученика убрать (M16 остаётся «кошельком» соларов, не источником рейтинга).
- **Предложение:** закрыть метрическую часть `OQ-RATING-01` в пользу Версии A; развести роли M3 (рейтинг ДНМ) и M16 (кошелёк/начисления).

## U2 — `REG-DNM-SOLARS` / `RATING` §2: разрез merit vs gift
- **тип:** CHANGE · согласуется с существующим proposal `earned⟂balance`.
- **Проблема:** сейчас солары и поднимают рейтинг, и тратятся в магазине → покупка формально снижает рейтинг; подарочные солары (ДР) некорректно влияют на успеваемость.
- **Предложение — два независимых разреза:**
  - **По источнику:** **merit-солары** (ДЗ, тесты, эссе, посещаемость, достиженческие значки) → **в рейтинг**; **gift/бонус** (ДР, промо, компенсация) → **только баланс**, в рейтинг не идут.
  - **По жизненному циклу:** **«заработано всего» (merit lifetime)** → рейтинг/матрёшка, не уменьшается при тратах; **«доступный баланс»** = merit + gift − потрачено → магазин.
- **Итог:** `рейтинг = посещаемость + своевременность ДЗ + merit-солары (lifetime)`; трата и подарки рейтинг не меняют.

## U3 — `REG-DNM-SOLARS`: 1190 = потолок ядра, merit-stretch сверх
- **тип:** CHANGE
- **Проблема:** жёсткий лимит 1190/семестр → добравшийся перестаёт стараться; полоса 1050–1190 узкая (~12%), рейтинг почти не различает.
- **Предложение:** **1190 = потолок ядра** (8 категорий core-программы); **merit-достижения сверх ядра идут в рейтинг ВЫШЕ 1190** (достиженческие значки, эссе, исследования, помощь одногруппнику, SENIOR — бизнес-проект). Под-лимиты по урокам в ядре. Это удерживает сильного ученика после ядра и разносит верх рейтинга. (Закрывает OQ §8 «в лимит или сверх» — **сверх**.)

## U4 — `SPEC-DNM-RATING-001` §8: подача тирами/перцентилем, не строгий топ
- **id:** §8 (`OQ-RATING-11`) · **тип:** CHANGE / RESOLVE
- **Проблема:** строгий лидерборд + узкая полоса/потолок → ничьи на максимуме; для «дна» демотивирует.
- **Предложение (что видит ученик):** **JUNIOR** — модель тиров (взял планку семестра → верхний тир; много победителей — норма) + прогресс-от-себя, без публичного «ты последний». **SENIOR** — перцентиль/относительное место + статус надёжности, соревнование **opt-in**. **Тай-брейк** (если нужен строгий порядок): стабильность → кто раньше достиг → качество (принятые ДЗ/эссе/проект) → иначе со-первые; никогда не «кто больше нагриндил».
- `[канон]` **«Матрёшка» вверх** (ученик→куратор→школа→франчайзи) остаётся **числовым lifetime-композитом** для агрегатов; ученику на экран — тиры/перцентиль. Одно не мешает другому.

## U5 — `SPEC-DNM-RATING-001` §2: сквозные срезы × когорта
- **id:** §2 (scope) + `OQ-RATING-07` · **тип:** CHANGE / RESOLVE
- **Канон сейчас:** scope ученика «группа · школа · сеть ДНМ (публичный)»; когортность сравнения не задана (`OQ-RATING-07`).
- **Предложение:** три среза одного рейтинга — **группа / школа / сеть (все школы всех франчайзи-партнёров)** — каждый **в разрезе когорты `JUNIOR|SENIOR × номер семестра`** (разная учебная нагрузка → честное сравнение, без нормировки в %). Межшкольные/сетевые срезы — **анонимизированы** (M16 BR-M16-012), пересчёт **раз в сутки 00:00** (FR-M16-009), рейтинг **не влияет на доступ** (BR-M16-013).
- **Детали:** `docs/superpowers/specs/2026-06-09-uchenik-skvoznoy-reyting-design.md`.

## U6 — `SPEC-M3` FR-M3-142 / `RATING` §8: ручные merit-значки + гардрейлы
- **id:** FR-M3-142 (6 авто-значков) + `OQ-RATING-06` (единый реестр), новый `OQ-RATING-12` (лимит) · **тип:** CHANGE / RESOLVE
- **Предложение:** к ≈6 авто-значкам добавить **ручные значки куратора** (идеальная работа, исследование, творчество, инициатива, помощь одногруппнику), несущие **merit-солары → в рейтинг (сверх 1190) и в баланс**. **Gift-значки/события (ДР)** — солары **только в баланс**. Пример: «идеальная домашка» = 100 соларов + очки в рейтинг.
- **⚠ Гардрейлы (критичны, т.к. влияют на рейтинг):** явные критерии каждого значка; согласованность между кураторами (анти-«любимчики»); видимость родителю; **лимит ручных merit-наград на период** (анти-инфляция рейтинга, `OQ-RATING-12`); фиксация в журнале начислений. Когорта: JUNIOR — коллекционные «собери все»; SENIOR — значки-статусы.

## U7 — `SPEC-DNM-TZ-001` §18.8: посещение (аддитивно)
- **тип:** ADD (поверх канона, инварианты посещаемости не меняются)
- **Канон сейчас:** посещение онлайн ≥40 мин (видеоконференция); непосещение → мгновенное уведомление родителю; решает администратор; ученик не отмечает.
- **Предложение (доказанная связка раннее-предупреждение + данные родителю):**
  - **Раннее предупреждение** по падению посещаемости/ДЗ — **до** «снежного кома», не только по факту пропуска (связка уже считается в «Успеваемости» — делаем триггером).
  - **Двусторонний канал родителю на языке семьи** (автоперевод) — родитель отвечает, не только получает.
  - **Tiered-реакция:** L1 nudge → L2 контакт куратора → L3 родитель/админ + разбор причины.
  - **Куратор — наставник посещаемости** (success mentor): пропуск = триггер раннего контакта. **Без автозаморозки** — приостановку (академ/каникулы) оформляет администратор с альтернативами.
  - **Когорта:** JUNIOR — стрик + командная цель; SENIOR — метрика надёжности + **плановый пропуск** (предупредил — взрослая норма, не штраф).
  - **Мягкий возврат** — запись урока, досдача, разбор причины вместо наказания.

## U8 — Удержание + когортный слой + мотивация-pull (аддитивно)
- **тип:** ADD (продуктовая подача; RBAC не меняется)
- **Удержание:** онбординг-первая-победа; re-engagement без вины («продолжи с того места»); передача гордости родителю; опора на СУ-привычки (21 день) и сертификат-ритуал.
- **Когортный слой представления:** одно ядро — две подачи (**JUNIOR Герой** / **SENIOR Основатель**); **ритуал перехода 13→14** (Герой → Профиль предпринимателя).
- **Посещение через мотивацию (pull):** принадлежность (главный драйвер) · предвкушение урока · прогресс-от-себя · автономия · признание присутствия · позитивный перелив (пиринг-наставничество). Принцип: предупреждение убирает причины не приходить, мотивация создаёт причины приходить.
