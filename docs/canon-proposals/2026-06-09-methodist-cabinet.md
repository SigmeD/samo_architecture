## M1 — Роль методиста (новая, ГО)
- **тип:** NEW ROLE · **код (провизорно):** `op-metodist-dnm`
- Методист головного офиса — оттачивает методику уроков, владеет учебной программой («ключевая роль рядом с продуктами/процессом», 08.06 §1). Ядро — «контур авторинга методики» (план → AI-сегментация → визуал видов → конструктор → утверждение руководителем → публикация кураторам).
- **Предложение:** внести роль методиста в `CONV-ROLES-DNM-001` + строку RBAC; домены — авторинг программы, конструктор видов, библиотека визуалов, цикл утверждения, методконтент Академии.

## M2 — Авторинг программы у методиста (три модели одного права)
- **тип:** CHANGE / RESOLVE (R2, BUG-08-02)
- **Канон RBAC v1.4:** «Конструктор программы (уроки/ДЗ/тесты)» = CRUD у руководителя `br7`.
- **Атлас (canon-proposal L1, lead-cabinet):** авторинг/редактор/публикация у **куратора франшиз**, у руководителя только gate-утверждение.
- **Встречи 08–09.06:** автор = **методист**; workflow «методист готовит → руководитель **утверждает/отклоняет, не правит**».
- **Предложение:** ратифицировать **методиста** автором; руководитель — gate (утв./откл.); снять авторинг с куратора франшиз (КФ переходит к курированию франчайзи, не методике). Обновить матрицу `/map` (ячейка «Программа·авторинг»): методист `full`, КФ — снять, руководитель — `утверждение`. **Решение владельца** — три актора надо свести к одному.

## M3 — Конструктор видов уроков
- **тип:** NEW
- Каждый **вид урока — свой визуал** (солнышки/шипы/фигурки), сложность считывается визуально. Виды: с ДЗ / без ДЗ / СЦ / пробный / групповой / индивидуальный. **AI-сегментация:** план уроков (описание+ДЗ) → AI раскладывает по видам → визуал → конструктор (добавил вид → автогенерация). Виды отражаются и в кабинете ученика (не одинаковые «кружочки»).
- **Предложение:** ввести сущность «вид урока» + конструктор в спеку методиста (новый раздел TZ/M3).

## M4 — Бизнес-академия = сквозной кабинет
- **тип:** NORM (BUG-08-09)
- Решение клиента: **сквозной кабинет, НЕ отдельный кабинет тренера** (08.06 §6/§7). Обучение по ролям (франчайзи/маркетолог/продавец/куратор/финансист), график по роли, темы, тренеры/эксперты, онлайн/оффлайн, **хранение записей**. В атласе смоделировано как сквозной слой (`overview.crossModules` усилен; домены «Академия» в кабинетах) — паттерн «функция, не кабинет» (как «отдел качества»). Кабинет тренера НЕ создаётся.
- **Предложение:** зафиксировать норму «Бизнес-академия — сквозная»; методконтент курса куратора («100 вопросов о дисциплине») готовит методист.

## M5 — Старший куратор ГО + правило связи
- **тип:** NEW ROLE (BUG-08-08, R8)
- Уровень «старший куратор ГО» над старшими кураторами школ. ⚠ Правило: старший куратор школы связан **только со старшим куратором ГО** + школа (админ/куратор/франшиза-директор), **НЕ напрямую с руководителем проекта**. В атласе: добавить роль в иерархию (сделано в `/map`); в `senior-curator.ts` убрать crossLink→lead (R8 — отдельная правка).
