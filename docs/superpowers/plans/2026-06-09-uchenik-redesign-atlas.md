# Перенос редизайна ученика в атлас (child.ts) — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Перенести редизайн кабинета ученика (merit/gift солары, рейтинг сверх 1190, 3 среза × когорта, тиры/перцентиль, удержание/посещение, когортный слой, split Эссе/БП, чат одноклассников) из черновика-дока в атлас `content/cabinets/child.ts` как контент-как-данные.

**Architecture:** Контент-как-данные. Все правки — в массивах `domains[]` / `sources[]` файла `child.ts` (строки-items, без новых полей схемы). Проверяемые точки расхождения с каноном помечаются `⚠` + ссылкой на `docs/canon-proposals/2026-06-09-uchenik-redesign.md` (канон ратифицируется отдельно через `propose_change`). Рендер — существующий дженерик `/cabinet/[slug]`, новых компонентов не нужно. Инварианты канона (ДЗ без баллов, тесты 70%/10, СУ≠проект, без финансов, нет «заморожен»/«7 этапов»/«0–100%») НЕ нарушаются.

**Tech Stack:** TypeScript strict, Zod (`content/schema.ts`), Vitest (`tests/content/child.test.ts`), Playwright screenshot-diff (`e2e/atlas.spec.ts`, golden `cabinet-child.png`).

**Источники намерения:** `docs/cabinets/uchenik-cabinet.md` (человекочитаемый редизайн), `docs/kabinet-uchenika-redesign.md` (наложение на канон), `docs/canon-proposals/2026-06-09-uchenik-redesign.md` (U1–U8), `docs/superpowers/specs/2026-06-09-uchenik-skvoznoy-reyting-design.md` (сквозной рейтинг).

**Итог по доменам:** 13 → **16** (split Эссе/Бизнес-проект = +1; +«Когортный слой»; +«Удержание и посещение»). Изменяются in-place: Главная, Геймификация, Рейтинг, Магазин, СУ, Сертификаты, Расписание/посещаемость, Чат, Профиль.

**Гард-инварианты (должны остаться зелёными во ВСЕХ задачах):**
- `JSON.stringify(child)` НЕ матчит `/заморожен/`, `/7 этап/`, `/0[–-]100%/`.
- Матчит `/без баллов/`, `/два самостоятельных блока/`.
- Все `crossLinks.toCabinet` резолвятся.
- Обезличено (нет имён) — guard `tests/content/impersonal.test.ts`.
- ⚠ Формулировку «без автозаморозки» НЕ использовать (близко к запрету `/заморожен/` по смыслу) — писать **«без автоматической приостановки»**.
