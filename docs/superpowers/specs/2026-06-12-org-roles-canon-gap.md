## 3. RBAC-дельта (атлас `/map` на v1.4 → канон v1.6)

`/map` матрица «роль×раздел×право» построена на `CONV-RBAC-DNM-001` **v1.4**. Канон сейчас **v1.6**.

**Таксономия (без изменений):** 8 нумерованных разделов-ресурсов + §9 «Маркетинговые ресурсы». Права: `—`/`R`/`CRU`/`CRUD`; скоуп-суффиксы `(св)`/`(дети)`/`(гр)`/`(шк)`/`(закр)`/`(все)`.

**v1.5 (11.06):**
- **+роль Мрк (Маркетолог)** 11-й колонкой. «Ноль финансов»: Мрк = `—` везде §1–§8, **кроме** §4 Рейтинги `R(св)`, §5 Чат `CRU`, §7 KPI `R(св)`.
- **+§9 «Маркетинговые ресурсы»**: лидоген/реклама `CRU(закр)`, SMM-аккаунт `CRU(закр)`, UTM `CRU`, доля квалиф. лидов `R`; **выручка/close rate = `—`** (только ГО).

**v1.6 (11.06, орг-синк 10.06):**
- **+5 ролей объявлены** (Директор школы, Финансист ГО, HR, Методист, Таргетолог) — **только high-level scope, БЕЗ §1–§8 колонок**; детальный per-resource RBAC отложен в **`OQ-ORG-03`** («после спек кабинетов»).
- **Finance-сплит:** `Фин` → **«Бухгалтер школы»** (P&L только своей школы, `CRU(шк)`); **сводный финблок по сети + роялти → Финансист ГО** (`R`-агрегат все школы).
- **Админ редефинирован → распределяющая/сервисная роль.** Управленческие функции (делегирование прав, фиче-тоглы, настройка KPI — §8) **изъяты у админа и отданы Директору школы**. Админ сохраняет операционку: §1 зачисление/перевод/заморозка `CRU(шк)`, §3 посещаемость/расписание `CRU(шк)`, §4 выдача товаров `CRU(шк)`, §6 приём оплат/рассрочка/дебиторка/возвраты, §7 табель. **P&L админу закрыт** (`—`).
- **Фрч редефинирован → «Франчайзи-партнёр (собственник)»** (владеет P&L/роялти; операционка вынесена Директору; в модели A владелец сам носит роль директора).

**РОП во ВСЕХ версиях — структурный узел, НЕ строка RBAC.** Гранты РОП школы = колонка «Мен»; РОП ГО/франшиз руководит ГО-подролями продаж.
