# Дизайн-спека: school-admin — рычаги «качество преподавания → рост набора» (P0)

> **Статус:** дизайн на утверждении (brainstorming → writing-plans). Источник — маркетинг-ресёрч лучших практик онлайн-школ (Workflow `wf_b1f98e1e-6c5`, 6 областей, ~50 практик с источниками) + проектный док «ЛК Администратор школы» + канон.
> **Дата:** 2026-06-07. **Скоуп (утверждён владельцем):** P0-пятёрка, якорь — кабинет `school-admin`, реципрокные зацепки в смежных кабинетах, канон-пропозалы где норматива нет.
> **Цель:** повышение качества преподавания → больше подключений НОВЫХ студентов.

## 1. Маркетинг-тезис (причинная модель)

**Качество урока → вовлечённость + измеримый результат ребёнка → удержание (COR) → высокий NPS родителя + соц.доказательство → рефералы/сарафан → набор НОВЫХ при низком CAC.** Франшиз-слой (отбор/QA/аудит) гарантирует работу цепочки в каждой точке.

**Ключевой инсайт:** сильнейший «реферальный» рычаг — не размер бонуса, а качество урока → NPS → органический сарафан (бюджет ≈ 0; Khan Academy; tutoring-школы растут 30–60%/год на сарафане). Бонусы только **усиливают** уже существующую удовлетворённость — включать поверх высокого NPS.

**Что у ДНМ уже есть (нацелить):** геймификация (солары/рейтинги/магазин), кабинет родителя, отдел качества (функция админа), видеофиксация уроков, пробный урок. **P0 — это операционализация существующего в петлю качество→набор + 3 недостающих движка (NPS, churn-warning, реферал).**

## 2. P0 — пять рычагов и их маппинг на атлас

### P0-1. Quality-loop: рубричный QA уроков + видеоразбор + коучинг куратора
- **Что:** расширить видеофиксацию из «сверки числа детей» в **оценку качества урока по рубрике** (структура/вовлечение/обратная связь) + видеоразбор + цикл коучинга педагога (методист/ст.куратор как coach).
- **Доказательство:** коучинг **+0.49 SD** качества, +0.18 SD результат (Kraft, Blazar, Hogan 2018, 60 каузальных иссл.); Тетрика — методист на всех уроках; Edthena — rubric-aligned наблюдение. ⚠️ caveat: коучинг хуже масштабируется → малые группы/выделенные наставники.
- **Атлас (school-admin):** обогатить домен **«🛡️ Отдел качества»** (рубрика+видеоразбор+коучинг-цикл); новый модуль `quality-coaching-loop`.
- **Реципрокно:** `curator` — личный рейтинг как KPI качества + получает рубрику/коучинг/цели (1 item / crossLink-label).
- **Канон:** REG-DNM-SCHOOL-ADMIN-001 §6 покрывает видеофиксацию; **рубрика+коучинг как норматив — кандидат на `propose_change`** (новый FR качества урока).
- **Метрика:** рубричный балл, % уроков с QA, eNPS куратора.

### P0-2. NPS-петля: промоутеры → реферал, детракторы → save-flow
- **Что:** авто-NPS родителя/ученика/куратора; **NPS 9–10 → авто-приглашение «приведи друга»**, детракторы → CAPA/save-call.
- **Доказательство:** parent NPS ↔ переподписка r=.29, value-for-money r=.66; education — топ-3 отрасли по NPS; tutoring 30–60% роста на петле промоутеров.
- **Атлас (school-admin):** обогатить **«🛡️ Отдел качества» / «📈 Отчётность»** (NPS-сбор/сегментация/CAPA); новый модуль `nps-loop`.
- **Реципрокно:** `parent` — отвечает на NPS + триггер «поделиться» (есть опросник каждые 4 урока = рейтинг школы → добавить NPS-слой); `sales/marketer` — закрывают промоутера в реферал.
- **Канон:** NPS как нормативная метрика — **кандидат на `propose_change`** (добавить в отчётность/рейтинги).
- **Метрика:** NPS (родитель/ученик/куратор), промоутер→реферал CR.

### P0-3. Save-call + ранние сигналы оттока
- **Что:** правила/модель на данных (пропуски, падение вовлечённости/time-on-platform, низкий отзыв, низкие солары) → флаг риска → **звонок директора в тот же день**.
- **Доказательство:** **save-call спасает 70%+** отмен; early-warning +20% к выпуску (Georgia State); churn-ML на attendance/engagement.
- **Атлас (school-admin):** обогатить **«📊 Дашборды AD»** → AD «Удержание/риск оттока» + save-call-алерты; новый модуль `churn-save-call`.
- **Реципрокно:** `curator` (получает триггер → педагогическая интервенция); `franchise` (retention/save-rate как KPI, эскалация критичных).
- **Канон:** churn-warning/save-call — операционная практика, в каноне норматива нет → честная пометка «best-practice / на согласовании», опц. FR.
- **Метрика:** save-rate отмен, lead-time сигнала, время реакции.

### P0-4. Реферальный движок «родитель→родитель» (учёт-якорь в админе)
- **Что:** двусторонний бонус (give-X-get-X) с **триггером после «события успеха»** + персональная ссылка/промокод + трекинг/атрибуция + антифрод.
- **Доказательство:** реф-клиенты **+16% LTV / −37% churn** (HBR); 25–35% набора из рефералов; CRM-автоматизация подняла реф-CR 2%→11%; Skyeng (10 000₽ + double-bonus 7 дней), Outschool ($50/$20).
- **Атлас (school-admin):** **учёт зачислений из рефералов + антифрод** в новом домене «🎯 Воронка набора и рефералы»; модуль `enrollment-funnel-referral`.
- **Реципрокно:** `marketer` (дизайн оффера/экономика/ссылки), `parent` (give-get кнопка + дашборд бонусов), `sales` (приоритетная обработка реф-лида).
- **Канон:** реферальная программа — **кандидат на `propose_change`** (новый FR).
- **Метрика:** k-factor, доля рефералов в наборе, CAC рефканала vs paid.

### P0-5. Воронка trial→paid (операционный обзор у админа)
- **Что:** админ как операционный центр видит воронку лид→пробный→договор→оплата; пробный = «первый-вин» урок; мультиканальный дожим (WhatsApp/ретаргет/промокод с дедлайном).
- **Доказательство:** trial = демонстрация продукта; action-before-signup +20% retention (Duolingo); дожим бот+менеджер ×1.7 (carrotquest); trial→paid >50% — хороший бенчмарк.
- **Атлас (school-admin):** новый домен **«🎯 Воронка набора и рефералы (trial→paid)»** — операционный обзор воронки + show-rate + дожим-статусы (+ реф-учёт P0-4); модуль `enrollment-funnel-referral` (общий с P0-4).
- **Реципрокно:** `sales` (пробный+закрытие+дожим), `curator` (designed first-win урок), `marketer` (лиды/ретаргет).
- **Канон:** пробный частично в SPEC-DNM-REG-001 / TZ §13.4; воронка-ops — пометить «на согласовании».
- **Метрика:** trial→paid CR (>50% хорошо), show-rate пробного.

**Канон-пропозалы (черновики после атласа, на ваше «да»):** новые FR на (1) рубрику качества урока + коучинг, (2) NPS-петлю в отчётности, (3) реферальную программу. Оформляются по образцу `samo-docs/docs/proposals/*.md`; в атласе до ратификации — `implStatus: planned` / «на согласовании».

## 4. Инварианты (не нарушать)
- Граница финансов админа (только дебиторка, не P&L/роялти/маржа) — реферальные бонусы/экономика считает marketer/бухгалтер, админ ведёт учёт зачислений.
- Оценивание ДЗ «принято/на доработку» без баллов — QA-рубрика про КАЧЕСТВО УРОКА (педагога), не про баллы ученику.
- ⚠️ Анти-паттерн «дрейф геймификации»: солары привязывать к учебному результату, не к «фарму» (Duolingo митигирует *Time Spent Learning Well*).
- Обезличенность (только роли); доля роялти 50% — нигде. Бенчмарк-числа — иллюстративные; нормативные числа ДНМ из канона.

## 5. Out of scope (P1/P2 — отдельная итерация)
P1: витрина результатов/портфолио (child/parent/marketer), прогресс-отчёт+баланс позитив/негатив (parent), habit-loop streak с 1-го урока (child/parent), trust-сигналы верификации (school-admin/sales), лид-магнит вебинар (marketer).
P2 (франшиз-слой): «тайный покупатель»+аудит точек, тест-драйв франшизы, cohort/sibling+амбассадорство.
