## 1. Роль и место в системе

**Куратор франшиз** (`op-kurator-franshiz-dnm`) — специалист, который **сопровождает и поддерживает франчайзи-партнёров** (владельцев школ) от лица франчайзера. Связующее звено «головной офис ↔ партнёры». Аналог «менеджера по работе с франчайзи / аккаунт-менеджера франшизной сети». Ведёт от нескольких до пары десятков точек.

**Главная задача:** убыточных/проблемных франчайзи переводить в успешные. Практик + мотиватор + управленец, **не наблюдатель с отчётами**. **Основной акцент кабинета — аналитика и дашборды.**

**Иерархия (определяет видимость данных):** Руководитель проекта/владелец (деньги, роялти, методика) → **Куратор франшиз ← ЭТОТ КАБИНЕТ** → Франчайзи-партнёр (директор школы) → Ст. куратор → Куратор → Админ школы → Родитель/Ученик. Сквозные помощники: маркетолог, РОП/продавец, финансист, эксперты.

**Разведение ролей (решение владельца 06.06, встречи 05.06/04.06):** куратор франшиз и **руководитель сети — РАЗНЫЕ роли**. «Руководитель сети» — это грань кабинета **франчайзи-партнёра** (директор «Франшизы моей школы», владеет сетью своих школ; 05.06 §94 «отчёт руководителя сети» — внутри кабинета франчайзи), НЕ данный кабинет. **Куратор франшиз — сотрудник головного офиса в ПРЯМОМ подчинении руководителю проекта** (кабинет руководителя проекта, `lead`); адресат отчётов по созвонам → руководитель проекта. *(Обезличено: только роли, без персональных имён.)*

**Зона:** `purple`. **implStatus:** `planned` (продукт роли не реализован в коде; дизайн опережает код).

## 2. Ядро-процесс «Цикл курирования франчайзи» (П1)

Лента, 6 шагов → повтор:
1. **Анализ** — дашборд/аналитика/рейтинги: выявить отстающих, проблемных, убыточных.
2. **Созвон** — по графику (общий еженедельно · по направлению раз в 2 недели · индивидуальный); участники все/школа/сотрудник.
3. **Фиксация** — отчёт по созвону (тема · что обсуждали · выводы) → руководителю сети; хранение записей.
4. **Действие** — задачи / подключение эксперта (план тем) / обучение / план вывода в плюс.
5. **Контроль динамики** — план/факт по месяцам, «было → стало».
6. **Развитие** — лучших отметить → транслировать опыт; проблемных → в зону курирования. → повтор.

*badge:* «курирует франшизы целиком · акцент на аналитику». *Вторичный поток (модуль):* «Запуск новой точки» — онбординг 30/60/90.

## 5. Модули (L2 бизнес-процессы) — кандидаты

- **Цикл курирования франчайзи** (П1) — `partial`.
- **Плановый созвон по направлению** (П2): выбор направления → создание созвона → авто-уведомление/ссылка/календарь → проведение/запись → отчёт.
- **Подключение эксперта** (П3): дефицит по аналитике → эксперт + план тем → курс через Бизнес-академию → расписание → контроль прохождения.
- **Работа с рейтингами** (П4): сбор → лучшие (мотивация/трансляция) / худшие (курирование).
- **Запуск новой точки**: чек-лист открытия 30/60/90.
- **Аудит стандартов**: шаблон → проверка → нарушения (фото) → план устранения → контроль.

## 10. Инварианты / границы (что НЕ делаем)

- НЕ дублируем кабинет франчайзи-партнёра: у куратора — **курирование и аналитика (чтение)**, не операционное управление школой.
- Расшифровка созвонов не автоматизируется — отчёт формирует куратор (ИИ — черновик).
- Финансы франчайзи — **только чтение** (для аналитики), не управление.
- **Обезличивание (правило владельца):** никаких персональных имён нигде — только роли. Доли роялти («50%» и т.п.) — НЕ показывать (memory).
- Генерик-термины владельца (точки/рецептуры/товарные остатки/поставщики) — адаптированы под edutech-контекст ДНМ (школы, кадры, расписание), сохранены как структурные функции (запуск/аудиты/операционка/тикеты/база знаний).
