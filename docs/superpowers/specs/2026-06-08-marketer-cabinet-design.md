## 2. Ядро = «Контур лидогенерации со сквозной атрибуцией»

Центральная ось (Google Doc 2): один лид = метка источника, не теряется до оплаты; событие конверсии = подтверждение оплаты бухгалтером → close rate обратно маркетологу.

1. 🎯 Кампания + UTM-источник — таргет/контекст/SMM/инфлюенсеры; метка {источник/кампания/креатив}
2. 📥 Лид с источником → CRM — UTM-автозахват в карточку; валидация/дедуп (R3)
3. ➡️ Передача менеджеру — лид с источником + норматив скорости
4. 💳 Событие конверсии (бухгалтер) — подтверждение оплаты = точка истины; лид «оплатил» с источником
5. 🔁 Close rate обратно — какие каналы дают ОПЛАТИВШИХ (не дешёвый CPL); петля качества с продажами
6. 🎛️ Оптимизация — бюджет в победителей; **KPI = квалиф. лиды / close rate** (не объём, R2)

**Note:** двухуровневая модель (HQ бренд/ассеты vs франчайзи регион/закреплённый аккаунт); R1 — без сквозного пайплайна атрибуции маркетолог слеп (на CPL/охватах) → canon-proposal.

## 4. Связи (направленные)

- **Менеджер по продажам** (`⇄`) — **передаёт лиды (UTM-источник)** → менеджер; получает **close rate по источнику/кампании** (петля качества). *(ключевая связка — взаимодействие с sales)*
- **Руководитель проекта / HQ** (`⇄`) — бренд/УТП/брендбук/готовые креативы/стандарты + маркетинговый фонд; HQ-маркетолог по проекту видит все аккаунты; отчёты/KPI вверх (двухуровневая модель)
- **Франчайзи/директор** (`←`) — бюджет, цели, региональная стратегия; взнос в маркетинговый фонд
- **Куратор франшиз** (`←`) — Академия, тренинги по маркетингу
- **Бухгалтер/финансист** (`←`) — **событие конверсии** = подтверждение оплаты (точка истины close rate); ROAS требует фида выручки (конфиденц., Фаза 3)
- **Администратор школы** (`⇄`) — ТК «оплата/договор/билет» — владелец админ; конверсия
- **Куратор** (`⇄`) — ТК «пробный урок» — владелец куратор; ТК-оценка → задача
