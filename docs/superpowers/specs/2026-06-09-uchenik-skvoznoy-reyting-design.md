## 1. Контекст и проблема

Канон `SPEC-DNM-RATING-001 v1.2` §1 описывает вертикальную «матрёшку» **ученик → куратор → школа → франчайзи** (агрегация снизу вверх — для управленцев). §2 для самого ученика уже задаёт **scope: «по группе · по школе · по сети ДНМ (публичный)»**, но:

- **`OQ-RATING-01` (открыт):** набор метрик ученика не сведён — Версия A (3 показателя: посещаемость + своевременность ДЗ + солары, `tz-dnm` §18.5 / `M3` FR-M3-150) против Версии B (5 показателей, `M16` FR-M16-009).
- **`OQ-RATING-07` (открыт):** «ключевые показатели матрёшки» не выписаны; **честность сравнения учеников разных когорт не специфицирована**.

В атласе домен «Рейтинг ученика» (`child.ts`) на момент задачи помечал состав рейтинга как «на подтверждении» и упоминал только лидерборд группы — школьный и сетевой срезы и когортность не были раскрыты.

## 2. Решения владельца (диалог 09.06)

1. **Состав рейтинга ученика подтверждён** = **Версия A**: посещаемость + своевременность ДЗ + заработанные солары. → метрическая часть `OQ-RATING-01` закрыта в пользу 3 показателей.
2. **Верхний охват** = «среди всех школ всех франчайзи-партнёров» — то есть **сетевой (network-wide)**, а не в пределах одной франшизы. Совпадает с каноном §2 «по сети ДНМ (публичный)».
3. **Когортность обязательна:** ученики сравниваются раздельно по когорте, потому что у когорт разная учебная нагрузка. Глубина когорты = **группа-бракет (JUNIOR/SENIOR) × номер семестра**.

## 3. Модель

### 3.1 Метрика (горизонтальный рейтинг ученика)
Единая мотивационная цифра ученика = **посещаемость + своевременность ДЗ + заработанные солары** (Версия A). Веса трёх слагаемых каноном не заданы — остаётся открытым (см. §6).

### 3.2 Лесенка охватов (3 среза одного рейтинга)
Один и тот же рейтинг ученика показывается в трёх разрезах-лидербордах:

| Срез | Кого сравниваем | Было |
|---|---|---|
| 👥 **Группа** | ученики своего куратора | есть (лидерборд группы) |
| 🏫 **Школа** | все ученики своей школы | **новое** |
| 🌐 **Сеть** | ученики всех школ всех франчайзи-партнёров (публичный) | **новое** |

Ученик переключает срез и видит **своё место + топ**. Это горизонтальные срезы уровня 1 матрёшки; вертикальная агрегация ученик→куратор→школа→франчайзи (уровни 2–4) — отдельный механизм для управленцев, не меняется.

### 3.3 Когортность (ядро честности)
Каждый срез считается **раздельно по когорте = {JUNIOR | SENIOR} × {номер семестра}**. Примеры лиг: «JUNIOR · семестр 1», «SENIOR · семестр 3». Ученик ранжируется только против своей когорты во всех трёх срезах. Новичок не конкурирует с ветераном — нагрузка одинаковая.

> Следствие масштаба: на старте точки в ячейке «когорта × школа» может быть мало детей — школьный срез информативен только при наполнении; сетевой срез когорты — основной для ранней мотивации.

### 3.4 Приватность и анти-читинг (мандат канона, не опционально)
- **Анонимизация детских рейтингов** (M16 BR-M16-012): межшкольный и сетевой лидерборд показывает герой-аватар + имя без фамилии + город/школу — не ФИО.
- **Пересчёт раз в сутки в 00:00** (M16 FR-M16-009) — не real-time, защита от накрутки.
- **Рейтинг не влияет на доступ** к обучению (M16 BR-M16-013).

### 3.5 Что видит ученик
В домене «Рейтинг ученика»: переключатель срезов (Группа / Школа / Сеть), внутри — своя когорта (бейдж лиги «SENIOR · семестр 2»), своё место, топ когорты, дельта «было → стало» при суточном пересчёте.
