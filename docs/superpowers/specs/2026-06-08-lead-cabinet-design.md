## 4. Связи (направленные)

- **Куратор франшиз** (`⇄`) — авторит учебную программу → руководитель подтверждает; брендбук/финмодель/онбординг-трекер у него; поддержка отстающих по FPI
- **Финансист/бухгалтер** (`←`) — ведёт финансы сети (источник истины) → руководитель видит **сводно** по правам
- **Франчайзи/директор** (`⇄`) — отчёты/KPI вверх; договоры; взнос в маркетинг-фонд; чат = эскалация/broadcast
- **Администратор школы** (`→`) — создаёт/курирует школы; целостность роялти (число детей)
- **Все роли** (`→`) — настройка прав (RBAC)
- **Куратор · Ученик** (`→`) — публикация программы вниз (через куратора франшиз)

## 6. Механизм «New» (item-level бейджи)

- Опц. флаг `isNew?: boolean` на `DomainSpec` / `ProcessStep` / `CrossLink` / `ModuleRef` (`content/types.ts` + Zod `content/schema.ts`).
- Компонент `NewBadge` (токены, без hex-литералов) — маленькая пилюля «🆕 New» в шапке домена/шага/связи/модуля **только при `isNew`**.
- `Legend` получает проп `hasNew` (страница вычисляет из спеки) → строка «🆕 New — добавлено в обновлении 08.06 (снимается при мердже в master)» рендерится только для кабинетов с новизной.
- **Аддитивность (критично):** прочие кабинеты не ставят `isNew` → их рендер байт-в-байт прежний → e2e-голдены не плывут.
- **База сравнения** = постер 02.06. **Снятие при мердже в master:** механический sweep по `isNew` (полный список новых блоков зафиксирован в canon-proposals/STATUS), документирован как обязательный шаг.
