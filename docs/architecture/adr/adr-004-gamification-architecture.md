# ADR-004: Архитектура геймификации ДНМ

**Статус:** ✅ Реализовано (2026-06-02)
**Заменяет план:** [ADR-003](adr-003-gamification.md) («перенос в Этап 4»)
**Документирует as-built.** Источник правды по состоянию — `docs/STATUS.md` §1.

## Контекст
ADR-003 откладывал геймификацию на Этап 4. По факту подсистема реализована раньше — этот ADR фиксирует её фактическую архитектуру, чтобы решение было прослеживаемо.

## Решение (как построено)
Геймификация разбита на отдельные backend-модули с собственными моделями:

| Подсистема | Модуль | Модели |
| --- | --- | --- |
| Валюта «Солары» | `solars` | `DnmSolar` (транзакции, баланс = SUM) |
| Достижения / уровни | `gamification` | `DnmAchievement`, `DnmStudentAchievement` |
| Рейтинги | `ratings` | `DnmAggregateRating`, `DnmRatingMetric`, `DnmRatingWeights`, `DnmCuratorRating` |
| Магазин за солары | `shop` | `DnmShopProductCatalog`, `DnmShopDistribution`, `DnmShopOrder` |

Ключевые принципы:
- **Начисление событиями.** Солары начисляются по событиям учебного процесса (тест с 1-й попытки на 100%, принятая практическая, эссе, отзыв родителя, бизнес-задача, сдача семестра в срок) + ручное начисление куратором с лимитами. Идемпотентность по `reason`.
- **Уровень** = `floor(solars / 100) + 1`; лидерборды по сумме соларов и метрикам за 30 дней.
- **Рейтинги** считаются для ученика / куратора / школы / франчайзи; веса метрик переопределяются по scope (GLOBAL / SCHOOL / FRANCHISE); есть dry-run.
