# ADR-001: ДНМ как автономный модуль

**Статус:** Принято (27.03.2026)
**Источник:** `docs/etap-3-dnm-plan.md` §1

## Контекст
Модуль LMS (Систематизация бизнеса) уже реализован в Этапе 2. Вопрос: расширять LMS для детского контура или делать отдельный модуль?

## Решение
ДНМ — автономный модуль с собственными таблицами (префикс `Dnm*`). Расширение LMS не требуется. Стек: React, Shadcn, Tailwind, NestJS.

## Обоснование
- Кардинально отличается бизнес-логика: семестры (не модули), возрастные группы, филиальная модель, оплата родителями.
- Собственная сущностная модель: `DnmSchool`, `DnmStudent`, `DnmSemester`, `DnmLesson`, `DnmHomework`, `DnmTest`, `DnmAttendance`.
- Общие абстракции (видео, тесты, ДЗ) могут использовать shared-библиотеки, но не наследовать от LMS-моделей.

## Последствия
- (+) Независимый релизный цикл, нет риска сломать LMS.
- (+) Проще адаптировать под детскую специфику (мультиязычность, расписание, посещаемость).
- (−) Возможное дублирование кода для media/video — решается через shared-модуль.
