# ADR-003: Геймификация — только ADR в Этапе 3

**Статус:** Принято (27.03.2026) · ♻️ **Пересмотрено (2026-06-02): реализовано** — см. [ADR-004](adr-004-gamification-architecture.md)
**Источник:** `docs/etap-3-dnm-plan.md` §1

## Контекст
Полная геймификация (Солары, уровни, достижения, «Путь ученика») запланирована в ТС для ДНМ.

## Решение (на 27.03.2026)
Вся реализация геймификации переносится в Этап 4. В Этапе 3 — только проектирование ADR.

## Обоснование
- Геймификация — отдельная подсистема, не блокирует базовый учебный процесс.
- Приоритет Этапа 3 — рабочий образовательный контур и управление школами.
- Модели должны предусмотреть точки расширения (`points` в `DnmStudent`, события для начисления соларов).

## ♻️ Пересмотр (2026-06-02)
Решение **развёрнуто по факту**: геймификация реализована в рамках расширенного Этапа 3, а не перенесена в Этап 4. В коде — модули `solars`, `gamification`, `ratings`, `shop` с реальной логикой и UI ученика (`/child/shop`, `/child/rating`, `/child/success`). Архитектура зафиксирована в [ADR-004](adr-004-gamification-architecture.md). Статус реализации — `docs/STATUS.md` §1.
