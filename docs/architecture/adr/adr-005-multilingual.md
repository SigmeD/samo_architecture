# ADR-005: Стратегия мультиязычности

**Статус:** ✅ Реализовано
**Источник плана:** `docs/etap-3-dnm-plan.md` (Эпик 10)

## Контекст
Контент платформы должен быть доступен на четырёх языках: **ru, kz, tj, uz**. Нужно выбрать механизм хранения и отдачи переводов (JSON-поля vs таблица переводов vs i18n-библиотека).

## Решение
- **Контент уроков** хранится в **JSON-поле** (`DnmLesson`), с ключами по языкам — перевод отдаётся по выбранному языку.
- **Интерфейс** — i18n на фронтенде (`frontend/public/locales/<lang>/*.json`).
- Язык выбирается при регистрации; интерфейс и учебные материалы показываются на нём.

## Обоснование
- JSON-поле для контента урока проще таблицы переводов при гибкой структуре блоков и не плодит JOIN'ы.
- Разделение «контент (БД, JSON)» и «UI-строки (locale-файлы)» — стандартная схема, независимые циклы обновления.

## Последствия
- (+) Гибкая структура мультиязычного контента урока без миграций под каждый язык.
- (−) Нет реляционных гарантий полноты перевода — контроль на уровне приложения/редактора.

## Утверждённый нейминг (важно)
Названия академии и форматов занятий регламентированы в корневом `CLAUDE.md` (раздел «Названия и нейминг»): только «Детская Бизнес Академия Саидмурода Давлатова»; форматы — «онлайн»/«офлайн» (не «очно»); валюта — «солары» (склоняется), не «соларс». Для kz/tj/uz — свои утверждённые формы.
