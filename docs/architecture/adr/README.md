# Architecture Decision Records (ADR) — ДНМ

Ключевые архитектурные решения модуля «Дети на миллион». Каждый ADR фиксирует **контекст**, **решение** и **последствия** на момент принятия. Решения не переписываются задним числом — при отмене добавляется пометка `Superseded by` / `Обновление`.

| ADR | Решение | Статус |
| --- | --- | --- |
| [ADR-001](adr-001-autonomous-module.md) | ДНМ — автономный модуль, не расширение LMS | ✅ Принято (27.03.2026) |
| [ADR-002](adr-002-parent-separate-account.md) | Родитель — отдельный аккаунт, не раздел в кабинете ребёнка | ✅ Принято (27.03.2026) |
| [ADR-003](adr-003-gamification.md) | Геймификация: в плане — «Этап 4» | ♻️ Пересмотрено → **реализовано** (см. ADR-004) |
| [ADR-004](adr-004-gamification-architecture.md) | Архитектура геймификации (солары, рейтинги, достижения) | ✅ Реализовано |
| [ADR-005](adr-005-multilingual.md) | Мультиязычность через JSON-поля контента (ru/kz/tj/uz) | ✅ Реализовано |
