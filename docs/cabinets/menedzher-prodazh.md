## Назначение

Кабинет менеджера по продажам ведёт **ТОЛЬКО воронку лида-родителя**: приём лидов от маркетолога, квалификация в CRM (анкета → рекомендация продукта + скрипты), выдача демо-доступа, запись на пробное, омниканальный дожим (WhatsApp ↔ CRM), получение согласия → передача администратору.

**Поправка к постеру:** **пробное проводит КУРАТОР** (менеджер только выдаёт демо-доступ и записывает); договор, оплату, рассрочку, наличную, дебиторку, возвраты и расторжение ведёт **АДМИНИСТРАТОР/ФИНАНСЫ**, оплату подтверждает **БУХГАЛТЕР** — менеджер только закрывает продажу до передачи и видит статус (read). Это расходится с `CONV-RBAC-DNM-001 v1.4` (Мен имеет CRU на приём оплаты и рассрочку, R на дебиторку; сводка даёт «проведение пробного урока») → canon-proposal.

**Границы (scope):** менеджер не видит общий раздел «Сделки» (только свою воронку); не редактирует продукты и программу; scope — только лиды-родители ДНМ. Продукты юрлицам/инвесторам — отдельная CRM Samo Global, вне этого кабинета. Speed-to-lead (первый контакт в минуты) — норматив и метрика. Обезличено (только роли).

## Ключевые инварианты (не нарушать)

- Менеджер ведёт **ТОЛЬКО воронку лида-родителя** — до передачи администратору, без оплат.
- **Пробное проводит КУРАТОР**, не менеджер (менеджер выдаёт демо-доступ и бронирует слот). ⚠ Канон-сводка RBAC v1.4 даёт пробное менеджеру → canon-proposal.
- **Договор, оплату, рассрочку, наличную, дебиторку, возвраты, расторжение** ведёт **АДМИНИСТРАТОР/ФИНАНСЫ**; менеджер не оформляет ничего из этого. ⚠ Расходится с `CONV-RBAC-DNM-001 v1.4 §6` (Мен: приём оплаты/рассрочка CRU, дебиторка R) → canon-proposal.
- **Оплату подтверждает БУХГАЛТЕР**; менеджер видит статус оплаты/зачисления только просмотр (read).
- Омниканальный дожим WhatsApp Business API ↔ CRM: вся переписка в карточке (аудит); история не теряется при уходе менеджера.
- **Speed-to-lead** (первый контакт в минуты) — норматив и метрика.
- Scope — только лиды-родители ДНМ; **B2B/инвесторы — отдельная CRM Samo Global**, вне кабинета.
- Обезличено (только роли).

## Домены и функции (9)

### 📊 Мои KPI · _только просмотр_
Воронка: лиды → демо → пробные → согласие → передано, с конверсией по этапам · Speed-to-lead — среднее время первого контакта (норматив) · активность (касания/звонки) — для коучинга, не наказания; цикл лид → согласие · конверсия в оплату — по статусу от администратора (read).
_Источник: CONV-RBAC-DNM-001 v1.4 §7 (KPI R(св)); проектный док §13._

### 📥 Лиды и приоритеты 🆕
Дневной список лидов по скорингу; «горячие» — первыми · источник лида (таргет/органика/реклама/рекомендация) — фиксируется · каденс касаний: 6–8 за первые 48 ч (звонки + сообщения) · гигиена воронки: флаги «протухших», авто-переназначение без реакции.
_Источник: проектный док §8/§16; REG-DNM-MARKETING-001 v1.1._

### 👤 Карточка клиента (CRM) 🆕
Клиент = родитель (физлицо): контакты + мессенджеры (WhatsApp/Telegram); менеджер добавляет, не меняет основной · входящий запрос вверху → анкета → рекомендация продукта → скрипты · отправка материалов одной кнопкой (афиша + презентация) · дубли: повторная заявка из «Потерянных» → карточка в «Новые», новая строка истории.
_Источник: SPEC-CRM-TZ-001 v1.1; проектный док §6. Без блока юрлица/B2B — это CRM Samo Global._

### 🎬 Демо-доступ и пробные уроки
Выдача демо-доступа родителю+ребёнку (ознакомительные уроки) · запись/слоты на пробное; ⚠ ПРОБНОЕ ПРОВОДИТ КУРАТОР (не менеджер) · коэф. успешности пробного (ориентир 70–80%) — у куратора · после пробного менеджер дожимает (омниканально).
_Источник: проектный док §17.1; встреча 04.06; SPEC-M3-DNM-001 v2.7._

### 📦 Карточка продукта · _только просмотр_ 🆕
Описание = скрипт презентации; афиша, расписание · цены в USD (раннее + стандарт × ВИП / Стандарт / Онлайн); конверсия по курсу ЦБ · скрипты презентации и возражений по продукту · менеджер не редактирует — только просмотр в рамках своего ранга.
_Источник: проектный док §7; SPEC-CRM-TZ-001 v1.1._

### 💬 Омниканальный дожим (WhatsApp ↔ CRM) 🆕
WhatsApp Business API интегрирован с CRM: вся переписка попадает в карточку (аудит) · единое окно: менеджер отвечает из CRM или WhatsApp — синхронно; история не теряется при уходе менеджера · шаблоны + журнал отправок; opt-in, вне 24-ч окна — только шаблоны; оферта дублируется почта + WhatsApp · WhatsApp — для лида во входящей воронке; платформенный чат — для уже на платформе.
_Источник: проектный док §9; ARCH-SERVICEMAP-001 v1.1._

### 🔁 Проигранные и прогрев 🆕
Закрытие «Проиграна» → обязательная причина из списка + комментарий · клиент → база прогрева; повторный обзвон не раньше 3 мес (маркетинг работает рассылками) · агрегация причин отказа → обратная связь маркетологу (какие источники конвертят).
_Источник: проектный док §12/§16._

### 📈 Статус по моим клиентам · _только просмотр_
Оплата/зачисление по своим клиентам — только просмотр (read) · ⚠ Договор, оплату, рассрочку, наличную, дебиторку, возвраты, расторжение ведёт администратор/финансы · оплату подтверждает бухгалтер; менеджер не видит общий раздел «Сделки» по сети.
_Источник: CONV-RBAC-DNM-001 v1.4 §6 (приём оплаты/рассрочка CRU у Мен — расходится с намерением); проектный док §10/§19 → canon-proposal._

### 🎓 Академия, задачи, связь
Обучающие созвоны (от куратора франшиз); скрипты презентаций/возражений; обучение по продуктам · задачи и уведомления (горячие лиды, перезвоны, протухшие) · чат с куратором/администратором (CRU); контакты.
_Источник: проектный док §14; CONV-RBAC-DNM-001 v1.4 §5 (чат с куратором/админом CRU)._

## Модули (бизнес-процессы, 8)

1. **Мои KPI (воронка)** — воронка лиды→демо→пробные→согласие→передано с конверсиями, speed-to-lead, активность (коучинг), цикл лид→согласие, конверсия в оплату по статусу админа. KPI R(св) — CONV-RBAC-DNM-001 v1.4 §7.
2. **Лиды, приоритеты, speed-to-lead** 🆕 — дневной список по скорингу, источник лида, каденс 6–8/48ч, флаги «протухших», авто-переназначение. speed-to-lead = норматив (минуты). Проектный док §8/§16; REG-DNM-MARKETING-001 v1.1.
3. **Карточка клиента (CRM)** 🆕 — родитель (физлицо), контакты+мессенджеры, входящий запрос, анкета→рекомендация→скрипты, дубли из «Потерянных»→«Новые». Без блока юрлица (B2B — Samo Global). SPEC-CRM-TZ-001 v1.1.
4. **Демо-доступ и пробные уроки** — ⚠ Поправка к постеру: менеджер выдаёт демо-доступ и записывает, ПРОБНОЕ ПРОВОДИТ КУРАТОР; коэф. успешности 70–80%. После пробного — дожим. Проектный док §17.1; встреча 04.06.
5. **Карточка продукта (скрипты)** 🆕 — описание=скрипт, цены USD (ВИП/Стандарт/Онлайн, конверсия ЦБ), афиша, расписание; менеджер не редактирует. Проектный док §7; SPEC-CRM-TZ-001 v1.1.
6. **Омниканальный дожим (WhatsApp ↔ CRM)** 🆕 — WhatsApp Business API ↔ CRM: переписка в карточке (аудит), единое окно, шаблоны+журнал, opt-in/24ч, оферта почта+WhatsApp. Снимает риск «дожим вне платформы». Проектный док §9; ARCH-SERVICEMAP-001 v1.1.
7. **Проигранные и прогрев** 🆕 — причина отказа (обязательно) → база прогрева (обзвон ≥3 мес), агрегация причин → обратная связь маркетологу. Проектный док §12/§16.
8. **Академия и обучающие созвоны** — обучающие созвоны от куратора франшиз, скрипты презентаций/возражений, обучение по продуктам, аналитика прохождения. Проектный док §14; CONV-RBAC-DNM-001 v1.4 §5.
