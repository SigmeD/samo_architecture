## Назначение

Кабинет франчайзи-партнёра (директора сети) — **ВЛАДЕЛЕЦ мини-бизнеса (1+ школ ДНМ)**, который платит роялти за владение и управляет своей сетью «как директор». Центр кабинета — **ДЕНЬГИ И ПЛАНИРОВАНИЕ, не операционка**: финрезультат сети (приход / уход / роялти 20% / чистая прибыль), планирование «от цели к плану» и контроль через расширенную аналитику.

По **СВОЕЙ** школе видит детальную финвыписку (P&L / Cashflow / ФОТ / дебиторка), по остальным своим школам — только сводные цифры; полный финотчёт видят лишь владелец и бухгалтер. Сам ставит план/факт (заполняемость, продажи) — он виден куратору франшиз для совместной работы; настраивает KPI-планы / проценты / пороги выплат и фиче-тоглы для своих сотрудников; видит рейтинги своей сети (школы / кураторы / продавцы / маркетологи) и своё место в рейтинге сети.

**Границы (scope):** строго своя сеть (`franchiseeId` из JWT). Операционку школы (расписание, приём оплат, табель, расторжение, выдача товаров) во избежание дублирования ведёт администратор школы — франчайзи входит «как директор» аналитикой поверх. Общесетевые цифры и распределение роялти живут у руководителя проекта — франчайзи их не видит.

## Ключевые инварианты (не нарушать)

- **Scope строго своя сеть** (`franchiseeId` из JWT); «все школы по проекту» — только у руководителя проекта.
- **Финансы ГО / бухгалтера — read-only**; по своей школе владелец видит детально, по остальным своим — сводно; полный финотчёт видят лишь владелец и бухгалтер.
- **Роялти 20% от ВАЛОВОЙ прибыли** отдельной строкой (OQ-M11-008 закрыт 07.06; код `samo-dnm` пока считает от выручки → реконсайл).
- **Доля распределения роялти НЕ показывается НИГДЕ** — живёт у руководителя проекта.
- **Не дублировать операционку администратора** — рутина у админа, франчайзи входит «как директор» аналитикой поверх.
- **«Магазин/распределение» и узел «Руководитель» исключены из цикла** управления сетью (решение 05.06).
- Обезличенность — только роли.

## Домены и функции (13)

### 📊 Дашборд владельца · _только просмотр_
Обзор сети: активные ученики, заполненность групп, оплаты за месяц, средняя посещаемость, дебиторка/просрочка · D1 Финансовый + роялти (MVP); D2 Рост и ученики / удержание / воронка / CAC-LTV (MVP); D3 План/факт «от цели к плану» (MVP) · D4 Операционное качество + целостность данных (фаза 2); D5 Карта + бенчмаркинг школ (фаза 2); D6 Персонал и ФОТ (фаза 2) · Алерты «Требует внимания»: дрейф KPI, отставание план/факт, убыточность (фаза 2).
_Источник: SPEC-FINANCE-REPORT-001 v1.3; SPEC-DNM-RATING-001 v1.2; проектный док ЛК (D1–D6)._

### 💰 Финансы и роялти (центр)
Две стороны: приход (оплаты учеников/родителей) и уход (расходы + ежемесячное роялти 20%) · По СВОЕЙ школе — детальная выписка (доходы/расходы/P&L/Cashflow/ОС/возвраты); по остальным своим школам — сводные цифры · Роялти = 20% от ВАЛОВОЙ прибыли ОТДЕЛЬНОЙ строкой (OQ-M11-008 закрыт 07.06: FINANCE-REPORT v1.3 / TZ §18.11; код `samo-dnm` пока от выручки → реконсайл); распределение доли владельцу идеи — НЕ здесь · Калькулятор роялти / финмодель — у бухгалтера в Excel, нигде не реализован; полный финотчёт видят только владелец + бухгалтер.
_Источник: SPEC-FINANCE-REPORT-001 v1.3; CONV-RBAC-DNM-001 v1.4 §6; SPEC-M11-FINANCE-001 v1.2 OQ-M11-008._

### 🎯 Планирование (план/факт)
Логика «от цели к плану»: желаемая годовая чистая прибыль → /12 → число учеников в постоянном обучении · Калькулятор макс. загрузки: узкое место — кураторы (число × часы/день × дни) → сколько детей пропустить (MVP, нигде не реализован) · Франчайзи САМ ставит план (год/месяц: заполняемость, продажи) → виден куратору франшиз; план vs факт по месяцам · Пороги роста сети: 30 / 60 / 90 / 120 / 150+ учеников (экономические вехи).
_Источник: SPEC-M10-FRANCHISING-001 v1.1; встреча 05.06._

### 🗺️ Сеть школ + карта
Матрёшка: франшиза → школы → группы → ученики → РОДИТЕЛИ (связка ученики→родители) · Карта региона с точками школ (адрес из карточки школы); фильтр «все школы по проекту / только мои» — фаза 2 · Scope строго своя сеть (`franchiseeId` из JWT); «все школы» — только у руководителя проекта · Оценка покрытия территории; владение несколькими школами (1+).
_Источник: SPEC-M10-FRANCHISING-001 v1.1; SPEC-M3-DNM-001 v2.7 §2.2; CONV-RBAC-DNM-001 v1.4 §1._

### 🏫 Управление школами («вход как директор»)
Вход в любую свою школу «как директор»: расширенная аналитика + все свойства администратора в цифрах · Настройка прав администратора, фиче-тоглы (вкл/выкл функций), делегирование прав = CRU(шк) · Создание/редактирование школы, зачисление/перевод/заморозка, расписание технически доступны франчайзи (FR-M3-091)… · …но операционная рутина делегирована администратору («зам по операционке»). «Вход как директор» = scope своих школ (`franchiseeId`) + права администратора через фиче-тоглы, БЕЗ отдельного режима/переключения контекста (FR-M3-094, 07.06).
_Источник: SPEC-M3-DNM-001 v2.7 FR-M3-091; CONV-RBAC-DNM-001 v1.4 §1,§8; SPEC-DNM-TZ-001 v3.8 §18.1._

### 🧑‍🎓 Ученики · Группы (реестры сети) · _только просмотр_
Реестры учеников и групп по всей своей сети (агрегат): наполнение, переходящие/новые · Привязка к родителям; контроль наполняемости групп против плана · Зачисление/отчисление/перевод доступны как директору, но операционно ведёт администратор · Без grade/баллов учеников — это контур куратора (инвариант ДНМ).
_Источник: CONV-RBAC-DNM-001 v1.4 §1; SPEC-M3-DNM-001 v2.7 §2.2._

### 👔 Персонал и оргструктура
Штатка (должность / оклад / часы / сумма к оплате) vs Орг-структура (рекомендательная схема подчинения) · Модель А (одна школа: франчайзи → бухгалтер + администратор → РОП/маркетинг + ст.куратор→куратор); Модель Б (сеть: отдел продаж под франчайзи, один бухгалтер на сеть) · Настройка KPI-планов / процентов / порогов выплат = CRU(шк); бухгалтер принимает на выплату = R · Просмотр KPI и табеля = R; перераспределение групп и замена заболевшего — у старшего куратора.
_Источник: ARCH-ORG-001 v1.2; CONV-RBAC-DNM-001 v1.4 §7; SPEC-KPI-PAYOUT-001 v1.1._

### 🎓 Академия обучения
Общая Академия всей цепочки персонала (менеджеры, маркетолог, таргетолог, РОП, продавец, ст.куратор, куратор, бухгалтер) · Две ветки: обучение франчайзи как предпринимателя / узкопрофильное обучение персонала · Общий календарь; франчайзи может сам пойти на любой тренинг (не ограничиваем) · Аналитика прохождения по школам / ролям.
_Источник: встреча 05.06; SPEC-M3-DNM-001 v2.7._

### 🏆 Отчётность и рейтинги · _только просмотр_
Ежемесячный отчёт франчайзи с 1 по 10 число → головной офис («Отправить отчёт») · Рейтинги своей сети: школы (по числу детей, возвратам), кураторы (лучшая пятёрка), продавцы, маркетологи · Своё место в рейтинге франчайзи сети (оборот, число учеников, расторжения −, доля кураторов с позитивом) · Бенчмаркинг школ между собой; динамика по периодам (фаза 2).
_Источник: SPEC-M3-DNM-001 v2.7 FR-M3-093; SPEC-DNM-RATING-001 v1.2 §6; SPEC-M10-FRANCHISING-001 v1.1 FR-M10-030/031._

### 🎥 Контроль целостности данных
Сверка числа детей видеофиксацией → защита роялти от занижения · Онлайн: видеоконференция фиксирует присутствие + запись; офлайн: камеры в классе → сжатый фрагмент / 3–4 фото на урок · Сверка отметок учителя с камерой (учитель отметил 7, камера 10 → вопрос франчайзи) · Готового техрешения в каноне нет — «для размышления» (на подтверждении).
_Источник: ARCH-SERVICEMAP-001 v1.1; SPEC-M11-FINANCE-001 v1.2 OQ-M11-008; встреча 05.06._

### 📣 Маркетинг региональный · _переключаемый блок_
Свой кабинет регионального маркетинга (аналитика по закреплённому аккаунту) — на согласовании · В рамках ЕДИНОЙ стратегии и брендбука ГО (нельзя продвигать сторонние услуги) · Распределение лидов по франчайзи приходит от таргета ГО (двухуровневая модель) · Бюджет / KPI маркетинга задаёт франчайзи; расходы на маркетинг — статья P&L.
_Источник: CONV-ROLES-DNM-001 v1.3 §2; REG-DNM-MARKETING-001 v1.1; встреча 05.06._

### 🛒 Магазин и распределение · _только просмотр_
Франчайзи видит ТОЛЬКО остатки мерча своей школы (R остатки шк) · Каталог / квоты / распределение по школам сети — у руководителя проекта (CRUD), не у франчайзи · Покупку за солары делает ученик; выдачу физтоваров — администратор школы · Исключён из цикла управления сетью (решение владельца 05.06) — оставлен отдельным справочным доменом.
_Источник: CONV-RBAC-DNM-001 v1.4 §4; SPEC-M3-DNM-001 v2.7 §2.2; встреча 05.06._

### 📋 Задачи · Документы · Коммуникации
Задачи и органайзер по сети; уведомления и алерты · Пакет франшизы как ПОЛУЧАТЕЛЬ: договоры, руководства, условия, паушальный взнос (read из пакета куратора франшиз) · Параметры экономики (паушальный / % роялти / пороги) конфигурируемы · Чат: эскалация конфликта от родителя франчайзи-партнёру (CRU шк); связь с ГО и куратором франшиз.
_Источник: SPEC-M10-FRANCHISING-001 v1.1 §7.7; CONV-RBAC-DNM-001 v1.4 §5._

| 🤝 **Менеджер по продажам** | ⇄ | Через школу: менеджер по продажам принимает оплату, ведёт график рассрочки, закрывает сделки (KPI Фикс+Бонус−Штраф). Франчайзи задаёт план продаж и % продуктов, видит воронку/лиды/договоры и выручку своей школы. |
| 👪 **Родитель** | ⇄ | Чат с родителем — эскалация конфликта на уровне администратора/владельца (CRU шк). Родитель может «Поблагодарить» франчайзи (вклад в его рейтинг сети); родитель — кандидат в воронку «Путь франчайзи-партнёра». |

## Модули (бизнес-процессы, 10)

1. **Калькулятор роялти / финмодель** _(divergent)_ — Роялти 20% ежемесячно отдельной строкой; база закрыта (OQ-M11-008, 07.06): «от валовой прибыли» (SPEC-FINANCE-REPORT-001 v1.3). divergent — код `samo-dnm` пока считает от выручки (реконсайл — задача разработки). Финмодель (виды договоров, учёт оплат) у бухгалтера в Excel — в каноне не описана, нигде не реализована. Распределение доли владельцу идеи НЕ показывается (инвариант).
2. **Планирование «от цели к плану»** — Желаемая годовая чистая прибыль → /12 → число учеников в постоянном обучении → план/факт по месяцам. Франчайзи сам ставит план (заполняемость, продажи) → виден куратору франшиз для совместной работы. Нигде не реализовано (намерение встреч 05.06).
3. **Калькулятор максимальной загрузки** — Узкое место — кураторы: число кураторов × часы/день × дни → сколько детей можно пропустить → связь с ценой → целевая сумма заработка. Намерение встреч 05.06, нигде не реализовано; нагрузка кураторов — у старшего куратора.
4. **Сеть школ и карта** — Матрёшка франшиза → школы → группы → ученики → родители; карта региона с точками школ (адрес из карточки, фильтр «мои/все»). Оценка покрытия территории. Карта — фаза 2, scope строго своя сеть (`franchiseeId` из JWT).
5. **Управление школой «как директор»** — Вход в любую свою школу с расширенной аналитикой + все свойства администратора в цифрах; фиче-тоглы и настройка прав администратора (FR-M3-091). Рутина делегирована администратору; механизм переключения контекста школы — на уточнении.
6. **Оргструктура и штатка (2 модели)** — Штатка (должность/оклад/часы/сумма) vs рекомендательная оргструктура; Модель А (одна школа) и Модель Б (сеть, один бухгалтер на сеть). Настройка KPI-планов/процентов/порогов выплат сотрудников (CRU шк); бухгалтер принимает на выплату (R).
7. **Академия обучения сети** — Общая Академия всей цепочки персонала (менеджеры, маркетолог, таргетолог, РОП, продавец, ст.куратор, куратор, бухгалтер) + ветка обучения франчайзи-предпринимателя; общий календарь, аналитика прохождения. Академия общая, не ограничиваем (05.06).
8. **Рейтинги и бенчмаркинг школ** — Рейтинги своей сети (школы по числу детей/возвратам, кураторы — лучшая пятёрка, продавцы, маркетологи); своё место в рейтинге франчайзи сети (оборот, ученики, расторжения −, кураторы с позитивом). Бенчмаркинг между школами — фаза 2.
9. **Контроль целостности (сверка числа детей)** _(divergent)_ — Защита роялти от занижения: видеофиксация уроков (онлайн — присутствие; офлайн — камеры → фрагмент/3–4 фото) + сверка отметок учителя с камерой (учитель 7, камера 10 → вопрос франчайзи). OQ-M11-008: «партнёры скрывают количество детей»; готового техрешения в каноне нет — «для размышления».
10. **Магазин и распределение (остатки)** — Франчайзи видит только остатки мерча своей школы (R); каталог/квоты/распределение по сети — у руководителя проекта (CRUD). Исключён из цикла управления сетью (решение владельца 05.06), оставлен отдельным справочным доменом.

## Открытые вопросы (показаны в атласе как «на подтверждении»)

- **Калькулятор роялти / финмодель:** источник логики калькулятора роялти и финмодели (у бухгалтера в Excel, в каноне не описан).
- **Контроль целостности:** техрешение сверки числа детей видеофиксацией (готового решения в каноне нет — «для размышления»).
- **Калькулятор загрузки:** реализация калькулятора макс. загрузки (нигде не реализован).
- **Маркетинг региональный:** объём регионального маркетинга (на согласовании).
- **Путь франчайзи-партнёра:** формат воронки «Пути франчайзи-партнёра».
- **Экономика:** номиналы (паушальный взнос / % роялти / пороги 30/60/90/120/150) — конфигурируемы, до ратификации ADR-0002 (Proposed).
- **База роялти:** код `samo-dnm` считает роялти от выручки vs канон «от валовой прибыли» (OQ-M11-008 закрыт — требуется реконсайл).
