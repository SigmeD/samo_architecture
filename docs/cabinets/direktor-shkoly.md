## Назначение

Кабинет директора школы ДНМ — управляющий школы (модель A: владелец-директор / модель B: наёмный директор); оверсайт + управление, операционка делегирована администратору. Scope = своя школа (`schoolId`). Покрывает полный операционный цикл: зачисление и путь ученика, учебный процесс и расписание, операционные финансы (приём оплат и дебиторка), коммуникацию с родителями как ядро, персонал/штатку/табель, отдел качества (видеофиксация уроков), отчётность и KPI школы, магазин-выдачу, документы.

Маркетинг, продажи (включая пробные уроки) и аналитика присутствуют в модели «одна школа» и выводятся под франчайзи в сетевой модели через пофункциональные feature-toggles.

**Жёсткая граница:** директор видит **только** операционные платежи и дебиторку — стратегические финансы (P&L, роялти, маржа) конфиденциальны и доступны только владельцу и бухгалтеру. Заморозка/приостановление/расторжение — только через директора/администратора, без автоматизма. Франчайзи-партнёр входит «как директор» (FR-M3-094): тот же операционный обзор плюс расширенная аналитика и право включать/выключать модули.

## Ключевые инварианты (не нарушать)

- **Scope строго своя школа** (`schoolId`).
- **Директор видит ТОЛЬКО операционные платежи и дебиторку** — стратегические финансы (P&L / ФОТ / роялти / маржа) конфиденциальны (только владелец + бухгалтер); роялти у директора НЕ показывается.
- **Заморозка / приостановление / расторжение / отчисление — ТОЛЬКО директор/администратор**, без автозаморозки, с альтернативами до приостановки.
- **Непосещение обрабатывает АДМИНИСТРАТОР** (не куратор).
- **Отдел качества = функция директора** (не отдельный кабинет/роль).
- **Пробные уроки — в «Продажи»** (не в учебном процессе).
- **Контент / ДЗ / солары — у куратора**, директор не оценивает.
- Адаптивность: 2 орг-модели (одна школа / сеть) через пофункциональные feature-toggles (не один рубильник A/B).
- Франчайзи входит «как директор» = scope собственных школ + права администратора через фиче-тоглы (FR-M3-094).
- Обезличенность (только роли); доля роялти 50% — нигде.

## Ядро: Зачисление ученика — мастер (Данные → Группа → Родитель → Договор → Оплата → Активен)

**Бейдж:** `ключевой процесс школы`

Зачисление = сквозная цепочка продаж, видимая директору в обратном порядке через Аналитику/CRM: источник лида → маркетинг/лидоген → менеджер (онбординг + пробное сам или назначает куратора) → дожим → ДОГОВОР оформляет менеджер → директор/АДМИНИСТРАТОР ПОДТВЕРЖДАЕТ → оплата через бухгалтерию (бухгалтер подтверждает) → активация → ст.куратор распределяет в группу. Оплата — единым блоком: онлайн-карта/СБП, наличные, QR/терминал, по счёту. Из карточки директор инициирует перевод между группами, приостановление, расторжение; заморозка/отчисление — ТОЛЬКО директор/администратор, без автозаморозки. Статус ученика: активен / не активен / ждёт оплаты.

## Домены и функции (14)

### 📊 Дашборды директора (AD1–AD6) · _только просмотр_
AD1 операционный: ученики всего/активные/новые/отток, посещаемость, заполняемость, KPI, блок «требует внимания» · AD2 набор и удержание: воронка лид → пробный → договор → оплата, план/факт наполнения, удержание · AD3 учебный процесс: посещаемость по группам/кураторам, кейсы непосещения · AD4 финансы-операции (НЕ P&L): платежи получено/ожидается, наличная-сверка кассы, дебиторка-просрочка, возвраты · AD5 персонал/качество: штатка/ставки, рейтинги/проверки кураторов, видеофиксация; AD6 магазин (заказы/остатки/выдачи).
_Источник: проектный dok ЛК «Администратор школы» §10; встреча 04.06._

### 🎓 Ученики и путь зачисления
Список учеников школы с фильтрами (группа, семестр, возрастная категория, статус); карточка ученика · единый экран ПУТИ ученика (воронка): источник лида → пробный → договор → оплата → активация · статусы ученика: активен / не активен / ждёт оплаты · из карточки: перевод между группами, приостановление, расторжение, заявки на доступ · реферальная регистрация — через родителя; за директором/администратором подтверждение оплаты.
_Источник: SPEC-M3-DNM-001 v2.7 §3.9 / FR-M3-080; SPEC-DNM-TZ-001 v3.10 §4.5._

### 📚 Учебный процесс и расписание
Типы занятий: групповые / индивидуальные / СУ (системы успеха) / доп.работы / бизнес-проекты / выезды · расписание всех групп: создание/перенос/отмена с автоуведомлением; шаблоны офлайн (сб-вс) / онлайн (2×нед) · посещаемость по группам: массовая отметка; статусы PRESENT/ABSENT/EXCUSED/LATE; онлайн засчитан при ≥40 мин · размер группы: индив. / 2–3 / до 5 или до 8 — лимит задаёт франшиза · граница: контент уроков, конструктор программы, проверка ДЗ и солары — у куратора, НЕ у директора.
_Источник: SPEC-M3-DNM-001 v2.7 §3.9 / FR-M3-081; SPEC-DNM-TZ-001 v3.10 §3.3, §4.5._

### 💳 Финансы школы (операционные — граница!)
Единый блок платежей: онлайн-карта/СБП · наличные · QR/терминал · по счёту; привязка платёж → договор/ученик/родитель/период; план платежа (рассрочка/график) · дебиторская и просроченная задолженность + напоминания; возвраты при расторжении (оформляет директор/администратор) · **ГРАНИЦА: директор видит ТОЛЬКО дебиторку / неоплаченные / операционные платежи** · P&L / ФОТ / Cashflow / роялти / маржа — конфиденциально, доступны только владельцу и бухгалтеру.
_Источник: SPEC-M3-DNM-001 v2.7 §3.9 / FR-M3-082; CONV-RBAC-DNM-001 v1.6 §6; SPEC-FINANCE-REPORT-001 v1.3 §5._

### 💬 Коммуникации / чат-хаб школы (ядро)
ЧАТ-ХАБ: директор — центральный узел прямых чатов школы. Прямые чаты с: Учеником (ТОЛЬКО старшие/SENIOR-группы; для младших/JUNIOR заблокирован — через родителя), Старшим куратором ДНМ, Родителем, Бухгалтером, Франчайзи, Менеджером по продажам, Маркетологом · единый двусторонний канал: чаты + видеосозвоны + история + модерация · 360-карточка ученика: профиль + переписка + статус оплаты + посещаемость + успеваемость на одном экране · мультиканал + автоперевод ru/kz/uz/tj; триггерные сообщения; обращения как тикеты с SLA; эскалация родитель → директор → франчайзи.
_Источник: проектный dok ЛК «Администратор школы» §4; встреча 04.06; дизайн вертикали/чат-хаба 2026-06-09._

### 👔 Персонал и оргструктура

_Источник: SPEC-M3-DNM-001 v2.7 §3.9 / FR-M3-083; SPEC-DNM-TZ-001 v3.10 §4.5, §18.12._

### 🛡️ Отдел качества (функция директора)
Не отдельный кабинет/роль — функцию исполняет директор школы · проблемные клиенты (удержание, эскалация); сбор обратной связи родителей (позитив/негатив) · по негативу — проверки кураторов; видеофиксация уроков (оценка качества куратора) · сверка числа детей по видео vs отметка учителя (целостность данных; расхождение → франчайзи) · рейтинг куратора (отзыв родителя — один из ВХОДОВ); план исправления по итогам проверки · **ГРАНИЦА: это ШКОЛЬНЫЙ аудит/эскалация по негативу; day-to-day педконтроль и коучинг кураторов — у СТАРШЕГО КУРАТОРА (см. senior-curator), не дубль.**

| 🏫 **Администратор школы** (`school-admin`) | ⇄ | ВНИЗ: операционка делегирована администратору школы (приём/подтверждение, оплаты-дебиторка, расписание/посещаемость, распределение, выдача за солары, табель). Директор видит аналитикой поверх и настраивает права/тоглы/KPI; администратор исполняет. |
| 🧑‍💼 **Старший куратор** (`senior-curator`) | ⇄ | ВНИЗ (учебная вертикаль): директор управляет старшим куратором школы — получает агрегированные отчёты по обучению/посещению и ОС кураторов, спускает методпакеты и KPI-планы. Day-to-day педконтроль и коучинг кураторов — у старшего куратора. |

## Модули (бизнес-процессы, 13)

1. **Зачисление ученика (мастер)** — 6 шагов: Данные → Группа → Родитель → Договор → Оплата → Активен. Сквозная цепочка продаж; менеджер оформляет договор, директор/администратор подтверждает, бухгалтер подтверждает оплату.
2. **Ученики и путь ученика** — список с фильтрами, карточка ученика, единый экран ПУТИ зачисления (воронка); статусы активен/не активен/ждёт оплаты; из карточки — перевод, приостановление, расторжение.
3. **Учебный процесс и расписание** — типы занятий (групповые/индивидуальные/СУ/доп.работы/бизнес-проекты/выезды), расписание по группам, посещаемость (≥40 мин онлайн), лимит группы от франшизы. Контент/ДЗ/солары — у куратора.
4. **Финансы школы (операционные)** — единый блок платежей (онлайн/наличные/QR/счёт), привязка платёж→договор/ученик, рассрочка, дебиторка/просрочка, возвраты. ГРАНИЦА: только операции; P&L/ФОТ/роялти/маржа закрыты (владелец+бухгалтер).
5. **Коммуникации / чат-хаб школы (ядро)** — чат-хаб директора (ученик-SENIOR, ст.куратор, родитель, бухгалтер, франчайзи, менеджер, маркетолог); единый канал: чаты + видеосозвоны; 360-карточка ученика; автоперевод; триггерные сообщения; тикеты с SLA; эскалация родитель→директор→франчайзи.
6. **Персонал и оргструктура** — штатка (должность/оклад/ставка-часы/сумма), оргструктура (рекомендательная), найм/табель кураторов из расписания. KPI→отчётность, качество→отдел качества. Настройка KPI-планов — у франчайзи/руководителя.
7. **Отдел качества (функция директора)** — не отдельный кабинет: проблемные клиенты, сбор ОС родителей, проверки кураторов по видеофиксации уроков, сверка числа детей (целостность), рейтинг куратора, план исправления.
8. **Отчётность и KPI школы** — отчёт директора (запись/пробные, оплаты+дебиторка, посещаемость, табель), еженед/месячные по шаблонам и направлениям, KPI школы, рейтинг куратора, явка/возврат; отчёты школа→головной офис. Финотчёты — конфиденц.
9. **Маркетинг / Продажи ⚙** — модель «одна школа»: лидоген, контент, Instagram школы; пробные уроки (менеджер проводит сам или назначает куратора). Под feature-toggle — в сети централизованы у франчайзи.
10. **Аналитика / CRM / план наполнения** — CRM-лиды (откуда пришёл ученик), собственный счётчик трафика, план/факт наполнения школы (напр. 250/120 — пример). Калькулятор «от цели к плану» виден франчайзи; директор наполняет до плана.
11. **Дашборды директора (AD1–AD6)** — AD1 операционный, AD2 набор/удержание (воронка + план/факт), AD3 учебный процесс, AD4 финансы-операции (НЕ P&L), AD5 персонал/качество, AD6 магазин. Франчайзи-директор — расширенный набор.
12. **Магазин (выдача за солары)** — базовая выдача физтоваров за солары канонична (TZ §18.6, FR-M3-084): директор/администратор отмечает выдачу, остатки уменьшаются. Каталог/статусы/квоты/антифрод (солары не за деньги) — расширение, на подтверждении.
13. **Адаптивность под орг-модель (A/B)** — модель A (одна школа): широкий кабинет — маркетинг+продажи+опер.бухгалтерия под директором. Модель B (сеть): продажи/маркетинг у франчайзи, бухгалтер один на сеть, директор/администратор ведёт учебный процесс. Реализация — пофункциональные feature-toggles. OQ-ADMIN-ORG.

## Открытые вопросы (показаны в атласе как «на подтверждении»)

- **OQ-ADMIN-ORG:** пофункциональный переключатель централизации A/B (одна школа / сеть) — две орг-модели через toggles «на школе / централизованно», не один рубильник.
- **KPI директора:** план/проценты (SPEC-KPI-PAYOUT-001 OQ); настройку KPI-планов задаёт франчайзи/руководитель.
- **Расширение магазина** (проектный dok §13): каталог/статусы заказа/квоты/инвентаризация/антифрод — на подтверждении.
- **Коэф. успешности пробного** (напр. 70–80% — на подтверждении).
- **Числа плана наполнения** (250/120 — пример, не норматив).
