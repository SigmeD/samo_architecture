## Назначение

Операционный кабинет администратора школы ДНМ — управляющий **одной школы**, «лицо школы»; в малой орг-модели исполняет функции директора. Scope = своя школа (`schoolId`). Покрывает полный операционный цикл: зачисление и путь ученика, учебный процесс и расписание, операционные финансы (приём оплат и дебиторка), коммуникацию с родителями как ядро, персонал / штатку / табель, отдел качества (видеофиксация уроков), отчётность и KPI школы, магазин-выдачу, документы.

Маркетинг, продажи (включая пробные уроки) и аналитика присутствуют в модели «одна школа» и выводятся под франчайзи в сетевой модели через пофункциональные feature-toggles.

**Жёсткая граница:** администратор видит **только** операционные платежи и дебиторку — стратегические финансы (P&L, роялти, маржа) конфиденциальны и доступны только владельцу и бухгалтеру. Заморозка / приостановление / расторжение — только через администратора/директора, без автоматизма. Франчайзи-партнёр входит «как директор» (FR-M3-094): тот же операционный обзор плюс расширенная аналитика и право включать/выключать модули.

## Ключевые инварианты (не нарушать)

- **Scope строго своя школа** (`schoolId`).
- **Админ видит ТОЛЬКО операционные платежи и дебиторку** — стратегические финансы (P&L / ФОТ / роялти / маржа) конфиденциальны (только владелец + бухгалтер); роялти у админа НЕ показывается.
- **Заморозка / приостановление / расторжение / отчисление — ТОЛЬКО админ/директор**, без автозаморозки, с альтернативами до приостановки.
- **Непосещение обрабатывает АДМИН** (не куратор).
- **Отдел качества = функция администратора** (не отдельный кабинет/роль).
- **Пробные уроки — в «Продажи»** (не в учебном процессе).
- **Контент / ДЗ / солары — у куратора**, админ не оценивает.
- Адаптивность: 2 орг-модели (одна школа / сеть) через пофункциональные feature-toggles (не один рубильник A/B).
- Франчайзи входит «как директор» = scope собственных школ + права администратора через фиче-тоглы (FR-M3-094).
- Обезличенность (только роли); доля роялти 50% — нигде.

## Ядро: Зачисление ученика — мастер (Данные → Группа → Родитель → Договор → Оплата → Активен)

**Бейдж:** `ключевой процесс школы`

Зачисление = сквозная цепочка продаж, видимая администратору в обратном порядке через Аналитику/CRM: источник лида (таргет / органика / реклама / рекомендация) → маркетинг/лидоген → менеджер (запись на пробный) → пробный урок (куратор как продавец) → договор-оферта → оплата → активация. Оплата — единым блоком: онлайн-карта/СБП, наличные («отметка наличной оплаты»), QR/терминал, по счёту; платёж привязан к договору/ученику/родителю/периоду. Из карточки админ инициирует перевод между группами, приостановление (академ/каникулы), расторжение; заморозка/отчисление — **только** админ/директор, без автозаморозки, с альтернативами до приостановки. Регистрацию ребёнка ведёт родитель по реф-ссылке без участия админа (масштабирование); за админом — подтверждение оплаты и заявки на доступ. Статус ученика: активен / не активен / ждёт оплаты («заморожен» в UI не используется). Цикл школы цикличен (постоянный набор, не учебный год): продажи → группа → запуск → отчёт директору → новый период.

## Домены и функции (14)

### 📊 Дашборды директора (AD1–AD6) · _только просмотр_
AD1 операционный: ученики всего/активные/новые/отток, посещаемость, заполняемость, KPI, блок «требует внимания» · AD2 набор и удержание: воронка лид → пробный → договор → оплата, план/факт наполнения, удержание · AD3 учебный процесс: посещаемость по группам/кураторам, кейсы непосещения · AD4 финансы-операции (НЕ P&L): платежи получено/ожидается, наличная-сверка кассы, дебиторка-просрочка, возвраты · AD5 персонал/качество: штатка/ставки, рейтинги/проверки кураторов, видеофиксация; AD6 магазин (заказы/остатки/выдачи).
_Источник: проектный док ЛК «Администратор школы» §10; встреча 04.06._

### 🎓 Ученики и путь зачисления
Список учеников школы с фильтрами (группа, семестр, возрастная категория, статус); карточка ученика · единый экран ПУТИ ученика (воронка): источник лида → пробный → договор → оплата → активация · статусы ученика: активен / не активен / ждёт оплаты («заморожен» в UI не используется) · из карточки: перевод между группами, приостановление, расторжение, заявки на доступ · реферальная регистрация — через родителя; за админом подтверждение оплаты.
_Источник: SPEC-M3-DNM-001 v2.7 §3.9 / FR-M3-080; SPEC-DNM-TZ-001 v3.8 §4.5._

### 📚 Учебный процесс и расписание
Типы занятий: групповые / индивидуальные / СУ (системы успеха) / доп.работы / бизнес-проекты / выезды · расписание всех групп: создание/перенос/отмена с автоуведомлением; шаблоны офлайн (сб-вс) / онлайн (2×нед) · посещаемость по группам: массовая отметка; статусы PRESENT/ABSENT/EXCUSED/LATE; онлайн засчитан при ≥40 мин · размер группы: индив. / 2–3 / до 5 или до 8 — лимит задаёт франшиза · граница: контент уроков, конструктор программы, проверка ДЗ и солары — у куратора, НЕ у админа.
_Источник: SPEC-M3-DNM-001 v2.7 §3.9 / FR-M3-081; SPEC-DNM-TZ-001 v3.8 §3.3, §4.5._

### 💳 Финансы школы (операционные — граница!)

_Источник: SPEC-M3-DNM-001 v2.7 §3.9 / FR-M3-082; CONV-RBAC-DNM-001 v1.4 §6; SPEC-FINANCE-REPORT-001 v1.3 §5._

### 💬 Коммуникация с родителями (ядро)
Единый двусторонний канал (замена внешним мессенджерам): чаты личные/групповые + видеосозвоны + история + модерация · 360-карточка ученика: профиль + переписка + статус оплаты + посещаемость + успеваемость на одном экране · мультиканал + автоперевод ru/kz/uz/tj; уведомления родителя (мгновенно/дайджест); Шаблоны ОС · триггерные сообщения: непосещение → связаться; просрочка → напоминание; пробный → follow-up; достижение → позитив · обращения как тикеты (приостановление/возврат/рассрочка/продление — статусы + SLA); эскалация родитель → админ → франчайзи.
_Источник: проектный док ЛК «Администратор школы» §4; встреча 04.06 (ядро «Коммуникация»)._

### 👔 Персонал и оргструктура

_Источник: SPEC-M3-DNM-001 v2.7 §3.9 / FR-M3-083; SPEC-DNM-TZ-001 v3.8 §4.5, §18.12._

### 🛡️ Отдел качества (функция администратора)

## Модули (бизнес-процессы, 13)

1. **Зачисление ученика (мастер)** — 6 шагов: Данные → Группа → Родитель → Договор → Оплата → Активен. Сквозная цепочка продаж (лид → пробный → договор → оплата → активация). Регистрацию ведёт родитель по реф-ссылке; за админом подтверждение оплаты.
2. **Ученики и путь ученика** — список с фильтрами, карточка ученика, единый экран ПУТИ зачисления (воронка); статусы активен/не активен/ждёт оплаты («заморожен» в UI не используется); из карточки — перевод, приостановление, расторжение.
3. **Учебный процесс и расписание** — типы занятий (групповые/индивидуальные/СУ/доп.работы/бизнес-проекты/выезды), расписание по группам, посещаемость (≥40 мин онлайн), лимит группы от франшизы. Контент/ДЗ/солары — у куратора.
4. **Финансы школы (операционные)** — единый блок платежей (онлайн/наличные/QR/счёт), привязка платёж→договор/ученик, рассрочка, дебиторка/просрочка, возвраты. ГРАНИЦА: только операции; P&L/ФОТ/роялти/маржа закрыты (владелец+бухгалтер).
5. **Коммуникация с родителями (ядро)** — единый двусторонний канал: чаты + видеосозвоны + история + модерация; 360-карточка ученика; автоперевод ru/kz/uz/tj; триггерные сообщения; обращения как тикеты с SLA; эскалация родитель→админ→франчайзи.
6. **Персонал и оргструктура** — штатка (должность/оклад/ставка-часы/сумма), оргструктура (рекомендательная), найм/табель кураторов из расписания. KPI→отчётность, качество→отдел качества. Настройка KPI-планов — у франчайзи/руководителя.
7. **Отдел качества (функция администратора)** — не отдельный кабинет: проблемные клиенты, сбор ОС родителей, проверки кураторов по видеофиксации уроков, сверка числа детей (целостность), рейтинг куратора, план исправления.
8. **Отчётность и KPI школы** — отчёт админа (запись/пробные, оплаты+дебиторка, посещаемость, табель), еженед/месячные по шаблонам и направлениям, KPI школы, рейтинг куратора, явка/возврат; отчёты школа→головной офис. Финотчёты — конфиденц.
9. **Маркетинг / Продажи ⚙** — модель «одна школа»: лидоген, контент, Instagram школы; пробные уроки (куратор как продавец, менеджер закрывает). Под feature-toggle — в сети централизованы у франчайзи.
10. **Аналитика / CRM / план наполнения** — CRM-лиды (откуда пришёл ученик), собственный счётчик трафика, план/факт наполнения школы (напр. 250/120 — пример). Калькулятор «от цели к плану» виден франчайзи; админ наполняет до плана.
11. **Дашборды директора (AD1–AD6)** — AD1 операционный, AD2 набор/удержание (воронка + план/факт), AD3 учебный процесс, AD4 финансы-операции (НЕ P&L), AD5 персонал/качество, AD6 магазин. Франчайзи-директор — расширенный набор.
12. **Магазин (выдача за солары)** — базовая выдача физтоваров за солары канонична (TZ §18.6, FR-M3-084): админ отмечает выдачу, остатки уменьшаются. Каталог/статусы/квоты/антифрод (солары не за деньги) — расширение, на подтверждении.
13. **Адаптивность под орг-модель (A/B)** — модель A (одна школа): широкий кабинет — маркетинг+продажи+опер.бухгалтерия под админом. Модель B (сеть): продажи/маркетинг у франчайзи, бухгалтер один на сеть, админ ведёт учебный процесс. Реализация — пофункциональные feature-toggles. OQ-ADMIN-ORG.

## Открытые вопросы (показаны в атласе как «на подтверждении»)

- **OQ-ADMIN-ORG:** пофункциональный переключатель централизации A/B (одна школа / сеть) — две орг-модели через toggles «на школе / централизованно», не один рубильник.
- **KPI администратора:** план/проценты (SPEC-KPI-PAYOUT-001 OQ); настройку KPI-планов задаёт франчайзи/руководитель.
- **Расширение магазина** (проектный док §13): каталог/статусы заказа/квоты/инвентаризация/антифрод — на подтверждении.
- **Коэф. успешности пробного** (напр. 70–80% — на подтверждении).
- **Числа плана наполнения** (250/120 — пример, не норматив).
