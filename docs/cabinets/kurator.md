## Назначение

Операционный кабинет педагога-куратора. Куратор — **не оценщик, а модератор/наставник**: проводит уроки во встроенном видеосозвоне по процедуре урока, проверяет ДЗ/классную работу (**принято / на доработку, без баллов**), даёт безоценочную обратную связь, вручную начисляет солары за субъективные работы, подтверждает посещаемость, формирует отчёты родителям, видит свой KPI/статистику, проходит обучение/тесты куратора.

**Границы (scope):** свои группы своей школы (`schoolId` + `groupId`). **Финансы / оплаты не отображаются.**

## Ключевые инварианты (не нарушать)

- **Куратор — модератор/наставник, не оценщик.**
- **ДЗ и классные работы оцениваются «принято / на доработку» — БЕЗ баллов** (никаких «0–100%», `grade`; ДЕТИ-7 от 21.04.2026).
- **Финансы / оплаты куратору не отображаются.**
- **Scope строго свои группы своей школы** (`schoolId` + `groupId`).
- **Обратная связь безоценочная** (принцип «позитив → зона роста → поддержка», техника «Бутерброд»).
- **Непосещение обрабатывает администратор школы, а не куратор.**
- **Отменённый «7-этапный» чек-лист процедуры урока заменён** конфигурируемой процедурой; финальный пакет процедуры передаёт руководитель проекта ДНМ.
- **Куратор не видит расчёты KPI других** и не настраивает планы KPI.
- **Баланс/рейтинг ученика — только просмотр**; магазин и выдача физтоваров куратору недоступны.

## Домены и функции (15)

### 👥 Мои группы
Список групп: семестр, число детей, ближайшее занятие · карточка группы и состав · перевод ученика в другую группу/время (автоуведомление админу).
_Источник: SPEC-DNM-TZ-001 v3.8 §4.4._

### 📋 Журнал занятий (процедура урока) · _переключаемый блок_
Чек-лист процедуры урока (`DnmCuratorChecklist`) · КАНОН: фиксированный «7-этапный» чек-лист отменён → конфигурируемая процедура (REG-DNM-LESSON-001 / 9-этапная инструкция) · привязка к расписанию и видеосозвону; фиксация посещаемости.
_Источник: REG-DNM-LESSON-001 v1.0; SPEC-DNM-TZ-001 v3.8 §4.4._

### 📤 Проверка ДЗ
Очередь по группам + фильтры (ожидают / проверены / на доработке) · статусы `PENDING → SUBMITTED → IN_REVIEW → ACCEPTED/REVISION` · решение «Принять» / «На доработку», комментарий обязателен; форматы текст / файл / фото · оценивание только «принято / не принято» — баллов нет (ДЕТИ-7 от 21.04.2026) · принцип ОС: позитив → зона роста → поддержка.
_Источник: SPEC-DNM-TZ-001 v3.8 §4.4._

### ✍️ Классные работы, эссе, бизнес-проекты
Три цикла проверки (ДЗ / эссе / бизнес-проект): свои статусы и начисление соларов · эссе семестра: приём + отметка публичного прочтения · бизнес-проекты: ревью одобрить / отклонить + ОС.
_Источник: SPEC-M3-DNM-001 v2.7._

### ✅ Посещаемость
Онлайн — автофиксация по подключению (≥40 мин), куратор подтверждает финальный список · офлайн — ручная отметка; статусы `PRESENT/ABSENT/EXCUSED/LATE` · непосещение обрабатывает администратор школы, не куратор.
_Источник: SPEC-M3-DNM-001 v2.7 §1._

### 📊 Успеваемость группы
Сводная таблица прогресса (уроки / тесты / ДЗ) с цветовой индикацией · статистика на главной: ученики / группы, проведённые часы · список вопросов классной / тестов и ответов учеников.
_Источник: SPEC-DNM-TZ-001 v3.8 §4.4._

### 📨 Отчёты родителям
Периодический отчёт по шаблону + ручная отправка, история · авто-отчёты присутствия (ежедн. / еженед. / ежемес.) — вручную только обратная связь · ежемесячный видео/голосовой отзыв (AI-генерация, куратор подписывает) · месячный отчёт старшему куратору.
_Источник: SPEC-DNM-TZ-001 v3.8 §18.9._

### 🪙 Геймификация: начисление соларов
Ручное начисление за субъективные работы — CRU(гр), 10–max по категории · авто-начисления (тест с 1-й попытки 100% и др.) идемпотентны, без куратора · баланс/рейтинг ученика — только просмотр; магазин и выдача физтоваров недоступны.
_Источник: SPEC-DNM-TZ-001 v3.8 §4.4; REG-DNM-SOLARS-001 v1.0._

### 📅 Расписание
Объединённый календарь уроков и видеосозвонов своих групп · отмена / перенос занятия (автоуведомления) · перенос ученика в другую группу/время.
_Источник: SPEC-DNM-TZ-001 v3.8 §4.4._

### 🎥 Видеосозвоны (конференции)
Все уроки (вкл. итоговый тест) — встроенный видеосозвон, без Zoom/YouTube · видео на собственном хранилище, скачивание ограничено · обязательная видеозапись офлайн-уроков.
_Источник: SPEC-M3-DNM-001 v2.7 §1._

### 📚 Все уроки / доступ к программе · _только просмотр_
Полный просмотр уроков и материалов своих групп без ограничения прогрессом · детальный просмотр урока с тестами/заданиями.
_Источник: CONV-RBAC-DNM-001 v1.4._

### 🎓 Обучение куратора / «Академия»
Тесты на знание программы (результаты видны владельцу/админу) — R(св) · методические материалы и видео-инструкции · канон v3.5: банк ~100 вопросов, аттестация с проходным баллом.
_Источник: SPEC-DNM-TZ-001 v3.8 §4.4._

### 📈 Мой KPI / статистика · _только просмотр_
Просмотр своих KPI: посещаемость / усваиваемость / в срок / пролонгации / отзывы · прогноз выплаты (Фикса + Бонус − Штрафы) · не видит расчёты других и не настраивает планы KPI.
_Источник: SPEC-KPI-PAYOUT-001 v1.1._

### 🧪 Пробные уроки
Расписание пробных уроков, коэффициент успешности · обязательная запись пробного урока · канон v3.5: пробный урок проводит куратор; менеджер закрывает сделку.
_Источник: SPEC-DNM-TZ-001 v3.8 §13.4._

### 📜 Сертификаты, документы, задачи, чат, уведомления
Сертификаты ученика = просмотр (процесс вручения) · документы куратора, поурочный план · задачи, уведомления; чат / видеозвонок с учеником / родителем / администратором.
_Источник: SPEC-DNM-TZ-001 v3.8 §4.4._

## Модули (бизнес-процессы, 14)

1. **Журнал урока / процедура онлайн-урока** — _divergent_ — Ядро-процесс. Код хардкодит отменённый «7-этапный» чек-лист; канон требует конфигурируемую процедуру (REG-DNM-LESSON-001).
2. **Проверка ДЗ** — _partial_ — Очередь + детальная проверка, статусы принято/на доработку, без баллов. Остаточное `grade(1..5)` в DTO противоречит отмене баллов.
3. **Классные, эссе, бизнес-проекты** — Три цикла проверки реализованы.
4. **Посещаемость** — Автофиксация ≥40 мин + подтверждение куратора; уведомления при `ABSENT`.
5. **Успеваемость группы** — Сводная таблица прогресса, статистика на главной.
6. **Отчёты родителям** — _partial_ — Отчёты по шаблону; авто-отчёты присутствия (ежедн./еженед./ежемес.) не реализованы.
7. **Начисление соларов** — _partial_ — Компонент есть; точка использования не верифицирована.
8. **Расписание и видеосозвоны** — Календарь + перенос занятия/ученика.
9. **Доступ ко всем урокам** — Каталог программы + детальный просмотр урока.
10. **Обучение куратора / Академия** — _partial_ — Training+квиз есть; полная «Академия» (банк ~100, аттестация) не подтверждена.
11. **Мой KPI** — _partial_ — `getMyKpi` есть; связь рейтинга с выплатами вне модуля curator.
12. **Рейтинг куратора** — _divergent_ — Код — только разрез школы; канон требует сетевой разрез + значок «Лучший куратор».
13. **Пробные уроки** — Соответствует канон-разрешению v3.5 (проводит куратор).
14. **Сертификаты** — Выдача сертификатов ученикам.

## Открытые вопросы

- **Ядро-процесс «на подтверждении»:** в каноне два кандидата структуры процедуры урока — 3-фазная (REG-DNM-LESSON-001) и 9-этапная методичка (REG-DNM-CURATOR-001); финальный пакет передаёт руководитель проекта ДНМ.
- **Журнал занятий:** фиксированный «7-этапный» чек-лист отменён каноном → конфигурируемая процедура (REG-DNM-LESSON-001 / 9-этапная инструкция).
- **Расхождение (divergent), модуль «Журнал урока»:** код хардкодит отменённый «7-этапный» чек-лист, тогда как канон требует конфигурируемую процедуру.
- **Расхождение, модуль «Проверка ДЗ»:** остаточное `grade(1..5)` в DTO противоречит отмене баллов.
- **Расхождение (divergent), модуль «Рейтинг куратора»:** код даёт только разрез школы, канон требует сетевой разрез + значок «Лучший куратор».
- **Незавершённые модули (partial):** авто-отчёты присутствия для родителей, точка использования начисления соларов, полная «Академия» (банк ~100 вопросов, аттестация), связь KPI/рейтинга с выплатами.
