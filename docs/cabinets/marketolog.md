## Назначение

Кабинет маркетолога ДНМ (`op-marketolog-dnm`) — **франчайзи-уровень**: региональная лидогенерация и передача лидов менеджеру по продажам. Ведёт региональный таргет и SMM на ассетах головного офиса (бренд / УТП / готовые креативы), собирает локальные лиды для своих школ.

**Двухуровневая модель:** HQ-маркетолог по проекту отвечает за бренд и видит все аккаунты сети; франчайзи-маркетолог (этот кабинет) — регион и аналитика только по закреплённому аккаунту.

Несущая ось — **сквозная атрибуция:** один лид = метка источника (UTM), которая не теряется до оплаты; событие конверсии = подтверждение оплаты бухгалтером → close rate возвращается маркетологу (какие каналы дают оплативших, а не дешёвый CPL).

**Границы:** маркетолог не закрывает сделку и не ведёт договор (это менеджер / администратор); не создаёт брендбук и не производит центральный контент (это HQ — он локализует ассеты в рамках бренда); не редактирует учебную программу и продукты; scope — школа (лагерь и франшиза вне scope). KPI = квалифицированные лиды / close rate, а не число лидов. Реклама — на родителя-плательщика и подростка SENIOR 14–17; младших — только через родителя. Обезличено (только роли).

## Ключевые инварианты (не нарушать)

- **Двухуровневая модель (HQ vs франчайзи):** HQ-маркетолог по проекту отвечает за бренд / контент и видит ВСЕ аккаунты сети; франчайзи-маркетолог (этот кабинет) — регион и аналитика только по **закреплённому аккаунту**.
- **KPI = качество, не объём** — квалифицированные лиды / close rate, а не число лидов (иначе мотивация тянет на объём, дашборд — на качество).
- **Передаёт лиды, НЕ закрывает сделку** — маркетолог отдаёт лид с источником менеджеру; договор / оплату / рассрочку ведут менеджер / администратор / бухгалтер.
- **Сквозная UTM-атрибуция** — лид-источник не теряется до оплаты; точка истины конверсии = подтверждение оплаты бухгалтером.
- **Не создаёт брендбук / центральный контент** — это HQ (видеограф / графдизайнер); маркетолог локализует готовые ассеты в рамках бренда.
- **Scope — школа**; лагерь и франшиза вне scope. Аналитика только по закреплённому аккаунту.
- **Обезличено** — только роли (имена из отчёта / дока убраны).

## Домены и функции (9)

### 📊 Дашборд маркетинга · _только просмотр_
Решающие метрики (CPL, лиды, close rate) — отдельно от диагностических (CTR/CPC/охваты/просмотры) · MK1 Лидогенерация: лиды по каналам, CPL по каналу/кампании, динамика · MK2 Эффективность: close rate по источнику (от продаж); бюджет план/факт; CAC/ROAS при фиде · MK3 Воронка: охват/трафик → формы → лиды → передано → оплата.
_Источник: проектный док §10._

### 🎯 Лидогенерация
Каналы: таргет, контекст, медийная, инфлюенсеры, органика (SMM), локальные мероприятия/партнёрства · источник лида (UTM-метка) фиксируется → в карточку CRM; менеджер видит «откуда пришёл» · передача менеджеру: лид с источником и контекстом, норматив качества данных · направление — ШКОЛА (зачисление детей); лагерь и франшиза — вне scope.
_Источник: проектный док §4; SPEC-CRM-TZ-001 v1.1._

### 📣 Реклама (Таргет)
Кампании по каналам с бюджетом; тест офферов и креативов на базе ассетов HQ (каждый вариант = своя UTM) · метрики: CPM, CTR, CPC, CPL (канал/кампания/объявление), конверсия лендинга · контроль бюджета (план/факт), перераспределение в эффективные каналы · ROAS — при наличии фида выручки из финансов (конфиденц., Фаза 3).
_Источник: проектный док §6; REG-DNM-MARKETING-001 v1.1 §5 (нормативы материалов)._

### 📱 SMM (аккаунт школы)
Аккаунт своей школы — изолированный бизнес-аккаунт, ведётся на шаблонах и в рамках бренда HQ · метрики: подписчики/рост, охват, просмотры (платное помечать), вовлечённость, действия в профиле, Direct · обращения из Direct → в воронку менеджера · бренд-консистентность держится на шаблонах/стандартах HQ; соцактивы → offboarding при отключении франчайзи.
_Источник: проектный док §5; REG-DNM-MARKETING-001 v1.1 §1.1/§4 (бренд-стандарты)._

### 🎨 Ассеты HQ и локализация
Библиотека готовых креативов головного офиса (видео, статика, шаблоны) · маркетолог адаптирует их под регион (язык, локальные детали) в рамках брендбука · ⚠ НЕ создаёт брендбук и НЕ производит центральный контент — это HQ (видеограф/графдизайнер) · нельзя продвигать сторонние услуги (напр. ментальную арифметику) — единый бренд.
_Источник: проектный док §7/§16; REG-DNM-MARKETING-001 v1.1 §3/§5._

### 🗺️ Точки контакта (ТК)
Карта ТК + оценка по 5-балльной; у каждой точки роль-владелец · низкая оценка точки → авто-задача в кабинет владельца (менеджер/куратор/администратор) · владельцы: реклама/лендинг — маркетолог; быстрый ответ/звонок после ПУ — менеджер; пробное — куратор; оплата — админ/бухгалтер · цикл: карта → оценка → задача владельцу → пересмотр (цель — все точки вызывают позитив).
_Источник: проектный док §8/§15.7._

### 💰 Бюджет и маркетинговый фонд
Региональный бюджет маркетинга (план/факт, утилизация) · маркетинговый фонд: взнос франчайзи и расходование на нац./регион. кампании · прозрачность «внёс / на что пошло» — связь с кабинетами франчайзи и руководителя.
_Источник: проектный док §9; связь с маркетинг-фондом руководителя._

### 📈 Отчётность · _только просмотр_
Еженедельно — каналы (CTR/CPC/CPL/конверсия) · ежемесячно — стратегия (close rate, бюджет) · ежеквартально — пересмотр KPI · структура майского отчёта: лидген по каналам (CPL, рост), SMM (до→после).
_Источник: проектный док §11; otchet-marketinga-may (non-canon, структура)._

### 🎓 Академия, двухуровневая модель, задачи
Обучение по маркетингу (digital, ТК, основы продвижения); тренинги от куратора франшиз · двухуровневая модель: HQ-маркетолог по проекту видит ВСЕ аккаунты сети (бренд); франчайзи — только закреплённый · «подчинение» относительное — ключевой критерий результат (охваты/просмотры/лиды) · задачи и уведомления (заявки, протухшие лиды, оценки ТК).
_Источник: проектный док §12; REG-DNM-MARKETING-001 v1.1 §1.1 (два уровня); встреча 05.06._

## Модули (бизнес-процессы, 9)

1. **Дашборд маркетинга** — решающие метрики (CPL/лиды/close rate) отдельно от диагностических (CTR/CPC/охваты). MK1 лидоген, MK2 эффективность (close rate), MK3 воронка. Проектный док §10.
2. **Лидогенерация** — каналы (таргет/контекст/медийная/инфлюенсеры/органика/локальные); UTM-источник → CRM; передача менеджеру с источником. Направление — школа (лагерь/франшиза вне scope). Проектный док §4.
3. **Реклама (Таргет)** — кампании+бюджет, тест офферов/креативов (UTM-варианты), CPM/CTR/CPC/CPL, контроль бюджета, ROAS при фиде выручки. Проектный док §6; REG-DNM-MARKETING-001 v1.1.
4. **SMM (аккаунт школы)** — аккаунт на шаблонах HQ; метрики (подписчики/охват/просмотры платные помечены/вовлечённость/Direct); Direct → воронка менеджера; бренд-консистентность; offboarding соцактивов. Проектный док §5.
5. **Ассеты HQ и локализация** — библиотека готовых креативов HQ; адаптация под регион в рамках брендбука. ⚠ Не создаёт брендбук/центральный контент (HQ). Нельзя продвигать сторонние услуги. REG-DNM-MARKETING-001 v1.1 §3/§5.
6. **Точки контакта (ТК)** — карта + оценка 5-балльная; у каждой точки роль-владелец; низкая оценка → авто-задача владельцу (менеджер/куратор/админ). Цель — все точки вызывают позитив. Проектный док §8/§15.7.
7. **Бюджет и маркетинговый фонд** — региональный бюджет план/факт + маркетинговый фонд (взнос/расходование, прозрачность «внёс/на что пошло»). Связь с франчайзи/руководителем. Проектный док §9.
8. **Сквозная атрибуция (несущая ось)** — единый лид-ID/UTM не теряется до оплаты; событие конверсии = подтверждение оплаты бухгалтером → close rate/CAC обратно. Без него маркетолог слеп (R1). Кросс-кабинет: маркетолог→менеджер→админ→бухгалтер. Проектный док 2 §0-§3 → canon-proposal.
9. **Академия и двухуровневая модель** — обучение по маркетингу; двухуровневая модель (HQ-маркетолог видит все аккаунты, франчайзи — закреплённый); критерий — результат. Проектный док §12; REG-DNM-MARKETING-001 v1.1 §1.1.
